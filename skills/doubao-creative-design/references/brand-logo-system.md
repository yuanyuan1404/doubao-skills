# Logo 系统

## 主 Skill 路由适配

本 ref 只提供 Logo 系统的产物规则。被 `doubao-creative-design` 路由调用时，工具协议、澄清轮次、核心资产记录和 prompt 总规则以主 `SKILL.md` 为准；本文件只补充 Logo 主版本、变体、规范页和文字门禁。

## 概述

Logo 系统是独立产物 skill，只解决品牌识别标志本身的建立、优化、变体延展和规范化，不生成无关 IP、包装、海报或完整 VI 手册。

Logo 必须先建立或确认一个核心图，即标准 Logo 主版本。理论上所有 Logo 变体都应基于该主版本调用 `image_edit` 延展，保持品牌识别一致。若用户只要求一个 Logo 主图，第一张必须是干净的标志主图；若用户要求 Logo 规范，再拆出组合、单色、小尺寸、网格或应用预览。

Logo 不只是单张标志图，后续视觉语言、包装和应用都应能追溯到它的图形基因。设计或提取 Logo 时，必须明确：

- 图形概念：品牌理念被转译成什么符号、几何或字形动作。
- 制图逻辑：圆切、直线、折角、负形、比例、对称或解构关系。
- 字标气质：衬线、无衬线、手写、宽体、窄体、拉伸、断笔或定制字形。
- 单色识别：纯黑、反白、小尺寸状态下仍能识别。
- 延展母题：可放大、重复、裁切、做纹样或做辅助图形的局部。

如果用户已提供 Logo 或强参考图，不要重新设计另一套无关 Logo；应从参考图中提取可延展的图形、色彩、字体和比例关系。

## 使用场景

当用户明确要求以下产物时触发：

- 品牌 Logo、标志、图形标、字标、标准组合。
- 横版 / 竖版组合、单色版、小尺寸头像、favicon。
- Logo 升级、Logo 规范页。
- 在已有 Logo 历史资产上要求延展、改字、换色、保持一致。

不应触发的情况：

- 用户要求完整 VI 全案、多章节品牌手册或多触点品牌方案时，应使用完整 VI 相关 skill。
- 用户只要求 IP、吉祥物、包装、海报、主 KV、门店、贴纸、书签或周边时，不使用本 skill。
- Logo 主图不生成无关吉祥物场景、复杂 mockup 环境、随机英文、假注册标、条码式细节或密集小字。

## 核心流程

本产物 ref 可独立参考。执行顺序固定为：理解用户意图 -> 确认历史资产和参考图角色 -> 建立或锁定核心 Logo 主版本 -> 制定产物张数与依赖 -> 为每张图写独立 `design_brief` -> 调用 `image_gen` 或 `image_edit` -> 结果加入图片引用表。被主 skill 路由时，工具字段映射以主 `SKILL.md` 的工具协议为准。

### 张数策略

除非用户明确要求完整 Logo 规范，通常生成 2-4 张。

- 轻量：1-2 张，适合单个 Logo 方向、字标优化、图形标优化或小尺寸头像。
- 标准：2-4 张，适合标准组合 + 图形标 / 字标 + 单色或小尺寸验证。
- 完整 Logo 规范：4-6 张，适合标准组合、横版/竖版、单色、小尺寸、制图规范和必要应用预览。

Logo 主图与 Logo 规范页必须拆开：主图只展示标志，规范页才展示网格、比例、色值和说明。

### 核心主版本

- 没有可用 Logo 锚点时，第一张先生成干净明确的标准 Logo 主版本，作为核心图。
- 有用户 Logo、历史生成 Logo 或强参考图时，先用 `image_edit` 建立或修正标准 Logo 主版本，不重新设计无关 Logo。
- 标准组合、图形标、字标、横版、竖版、单色、小尺寸、favicon、规范页和必要应用预览，原则上都以核心主版本作为参考图调用 `image_edit` 生成。
- 同一轮多图必须复用同一套 `image` 参考图顺序和锁定变量；如果某张图不能传参考图，需要在 prompt 中完整复写锁定变量。

### 一致性与多样性

独立运行时也必须先锁定上游核心锚点，再做差异化延展。不要为了多样性破坏品牌识别，也不要为了一致性让多张图只像同一画面的轻微改版。

一致性锁定：

- 锁定用户明确提供或历史生成确认的品牌名、Logo 图形、字标拼写、主色、字体气质、材质、图形母题、画面文字和关键语气。
- 所有可见文字必须逐字保留用户提供内容，不自行更换品牌名、活动名、产品名、口号或数值信息。
- 锁定品牌名拼写、图形几何、线条粗细、转角逻辑、字标气质、配色和间距。变体只能调整布局、比例、间距和色彩模式。

多样性释放：

- 多图之间至少在组合方式、色彩模式、版式比例、应用场景、规范信息层级或小尺寸验证方式中选择 2-3 项形成真实差异。
- 差异必须服务用户意图和 Logo 用途；不要只换背景、只换角度或只换装饰。
- Logo 系统的多样性只能来自标准组合、图形标 / 字标、横版 / 竖版、单色、小尺寸和必要应用预览；Logo 主图不得用复杂场景、长文或随机符号制造丰富感。

### Logo 主图边界

Logo 主图不是 Logo 规范页。标准组合 Logo、图形标、字标、横版/竖版组合的主图画面只展示标志本身。

Logo 主图只允许出现：

- 图形标；
- 字标 / 品牌名；
- 必要的中英文品牌名组合；
- 极少量用于标志组合本身的短副标，如果用户明确提供。

设计理念、方案编号、版本序号、制图网格、比例说明、色值、应用规范、使用规则、设计思路和说明段落都不属于 Logo 主图内容。用户要求展示这些信息时，拆成独立的 Logo 规范页或品牌手册页，不要塞进 Logo 主图。

Logo 主图 prompt 应写成正向纯展示结构，例如：`白色或浅灰纯净背景，中央展示标准组合 Logo，左侧为简洁图形标，右侧为品牌名“{品牌名}”，整体水平居中，周围留出充足安全区。`

Logo 规范页才可以包含网格、比例、单色版、小尺寸测试和说明标签。

### Logo 规范页文字门禁

Logo 规范页默认属于文字高风险场景。生成前必须先在内部建立 `visible_text_inventory`，最终 prompt 只能渲染清单内文字，没有进入清单的说明、注释、色值、规则和标签都不能写进画面。

默认可见文字优先控制在 5-8 处，复杂规范页最多 12 处。用户没有提供具体说明文案时，只允许使用版本标签，不生成解释句、自然段或泛化说明。

Logo 规范页常用标签应逐字明确，例如：

- “标准组合”
- “横版组合”
- “竖版组合”
- “图形标”
- “字标”
- “单色版”
- “小尺寸”
- “安全区”

每一处文字必须和对应容器绑定描述，同时写清字号层级和字体风格。例如：`下方四个等宽模块分别展示横版、竖版、图形标和字标，每个模块底部显示中号冷灰色几何无衬线标签“横版组合”“竖版组合”“图形标”“字标”。`

禁止在 Logo 规范页 prompt 中使用会诱导模型自由补字或生成模糊小字的表述，包括 `简短说明文字`、`小字说明`、`下方标注说明`、`说明文字`、`规范说明`、`应用场景说明`、`相关说明`、`若干说明`、`辅助信息`、`密集小字`、`极小文字`、`示意文字`、`色值示意文字`。

错误示例：`每个版本下方标注简短说明文字。`

正确示例：`每个版本下方只显示一行中号几何无衬线标签，文字依次为“标准组合”“横版组合”“竖版组合”“图形标”“字标”，不添加解释句。`

### Prompt 策略

每张 `design_brief` 都要先写生成目标，再写从参考图提取的 Logo 锚点，最后按目标画面结构描述新图。不要只写“参考原图风格”。

`image_gen` 时，先建立一个干净明确的 Logo 方向。

`image_edit` 时，只提取可复用的 Logo 锚点：

- 品牌名准确拼写；
- 图形概念；
- 线条和转角逻辑；
- 配色/材质；
- 间距气质；
- 字体方向。

不要过度保留参考图的页面背景、裁切、场景道具、阴影或偶然版式，除非它们本身就是 Logo 系统的一部分。

每张 Logo prompt 都要有明确差异：标准组合、图形简化、字标版、横版、竖版、单色转换、小尺寸验证。

所有画面文字必须加中文双引号。如果只出现 Logo 文字，就只列品牌名。

## 输入

- 用户提供的品牌名、产品名、人名、活动名必须逐字保留。
- 用户提供的 Logo、历史生成 Logo、IP、包装、主 KV、视觉语言锚点或强参考图，应作为 `image_edit` 参考图传入。
- 多参考图必须在 prompt 中按 `[img0]`、`[img1]`、`[img2]` 逐张说明作用，编号与 `image` 入参顺序一致。
- 价格、销量、折扣、功效百分比、排名、评分、认证编号、监管声明等数值或权威信息只能来自用户提供内容或已确认资产。
- Logo 规范页、高密度信息物料和带说明标签的页面必须先建立 `visible_text_inventory`；最终 prompt 只能渲染清单内文字。没有正文时，用版本标签、短标题、短标签、数字、步骤名和图标卡片表达。

## 输出形式

可选产物：

- 标准组合 Logo：图形 + 品牌名。
- 横版/竖版组合。
- 单独图形标或单独字标。
- 黑白单色版本。
- 小尺寸头像 / favicon 验证。
- Logo 规范页。
- 必要时增加一张 Logo 应用预览。

所有正式产物必须加入图片引用表。不要把多个正式产物合并到同一画布；Logo 主图和 Logo 规范页必须拆开。

## 工具与脚本

- `image_gen`：只在没有用户参考图、历史生成图、核心锚点，且必须建立第一张核心 Logo 主版本时使用；入参为独立 `prompt` 和 `ratio`。
- `image_edit`：用于已有用户图、历史图、Logo / IP / 包装 / 主 KV / 视觉语言锚点的延展；也用于基于核心 Logo 主版本生成所有 Logo 变体。入参为独立 `prompt`、`ratio` 和按顺序传入的 `image` 数组。

工具调用规则：

- `prompt` 只放完整 `design_brief`，不写比例、工具参数、文件路径、URL、模型名、seed、输出文件名或内部 skill 名称。
- `design_brief` 必须完成三层推演：载体骨架、场景语义、实例填充。复杂画面按物理空间分区扫描，每个可见元素写清位置、类型、形态、内容、文字样式、状态、关系、风格和工艺。
- `image_edit` brief 先写生成目标，再写参考图提取项，最后展开目标画面；至少 70% 内容描述目标图，不复述旧图。旧背景、旧裁切、旧文字、偶然道具和噪声只做内部判断，不写进最终 prompt。
- 所有画面文字必须逐字明确，并用中文双引号包裹；每处文字必须写清字体大小和字体风格。
- 不使用 `xx`、`某某`、`品牌名`、`对应色值`、`应用场景说明`、`相关说明`、`规范说明`、`说明文字`、`标签文字`、`若干文字`、`辅助信息`、`短说明`、`简短说明文字`、`小字说明`、`下方标注说明`、`自然段正文`、`介绍文案`、`补充信息`、`文字占位` 等会让模型自由补字的表述。
- 批量 / 并发仅限同参考图集合、同 `image` 顺序、同 `ratio`、互不依赖且每张都有独立 `design_brief` 的衍生产物。
- 传入工具前自检：prompt 中没有负向提示、风险词、工具字段、路径、模型名、画幅参数；所有中文双引号内容都是最终画面文字；重复结构没有用“等”“若干”“同上”省略；Logo 规范页已建立 `visible_text_inventory`，且没有 `极小文字`、`密集小字`、`示意文字`、`简短说明文字` 等不可读文字描述。
