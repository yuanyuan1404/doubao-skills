# 信息图与长图设计生成器

## 主 Skill 路由适配

本 ref 只提供信息图、图表化说明和结构化长图的产物规则。被 `doubao-creative-design` 路由调用时，工具协议、澄清轮次、核心资产记录、`image_gen` / `image_edit` 字段和 prompt 总规则以主 `SKILL.md` 为准；本文件只补充信息结构、屏幕职责、图表表达、长图拆分和文字可读性规则。

## 概述

用于把文案、大纲、数据、流程、对比关系或说明材料转换为结构化视觉图。核心原则是先保证信息层级、阅读路径、事实边界和文字清晰度，再追求视觉风格。

长图类任务默认拆分为多张 3:4 顺序图，除非用户明确要求单张长图、一图到底、不分页、固定极长画布或极端长宽比。

## 使用场景

**何时触发：**

- 用户明确提到“信息图”“infographic”“一图读懂”“长图”“多屏长图”“知识长图”“科普长图”“数据长图”“流程长图”“对比长图”“图表”“图解”“可视化说明”。
- 用户要求把一长段文案、内容大纲、说明材料、数据指标、设计规格或方法论转换为图片。
- 用户需要基于已有版式图、产品图、品牌资产或上一轮生成图，延展一套连贯的说明性图。

**何时不触发：**

- 强电商销售属性、平台商品图、淘宝 / 亚马逊 / A+ / 商品详情页屏次 / 主图 / 卖点图任务，优先走 `ecommerce-design.md`。
- 品牌 H5、落地页、活动页、宣传册、品牌触点长图等以品牌应用载体为核心的任务，优先走 `brand-application-system.md`。
- 社媒平台封面、笔记配图、短视频封面或平台传播图，且重点是平台视觉风格而非信息结构，优先走 `media-creative.md`。
- 纯文字翻译、纯代码编写、UI 前端代码生成。

## 输入与澄清

需要判断四类输入：主题 / 核心内容、用途 / 交付场景、目标受众、风格调性。

- 澄清最多一轮，只问最影响结果的问题。
- 信息轻微模糊时不追问，内部选择最贴近意图的方案。
- 缺失 2 项及以上时只问一个问题，优先问用途，其次问主题。
- 用户说“你来定”时，按输入内容推断用途与风格，直接执行。

短问题模板：

- 用途问：`这张信息图主要用于哪类场景？A. 社交传播 B. 长文说明 C. 活动 / 落地页转化`
- 风格问：`更倾向哪种视觉方向？A. 极简清晰 B. 精致商业 C. 数据专业`

事实与数据边界：

- 用户未提供具体数据、来源、价格、排名、认证、收益、医学功效或监管声明时，不得编造为事实。
- 可以用低风险的说明性占位文字，例如 `“来源待补充”`、`“数据待补充”`、`“二维码占位”`，但不能伪造官方 Logo、证书编号、认证章或真实机构背书。
- 若用户授权“可合理假设”，仍需避免医疗、金融、功效、监管和排名类高风险数据。

## 张数与画幅策略

优先级：用户明确要求 > 本 ref 策略 > 主 `SKILL.md` 默认张数。

- 单张信息图、单个流程图、单个对比图、单页图表说明：默认 1 张。
- 长图类任务未指定单张时，默认拆分为多张 3:4 顺序图。
- 轻量内容：3 张，适合单主题、少要点、简单流程。
- 标准内容：5 张，适合背景、痛点、核心观点、方法、总结。
- 复杂内容：7-9 张，适合多章节、数据证明、对比矩阵、流程和总结并存。
- 用户明确“一张长图”“不要拆分”“不要分页”“一张到底”“一镜到底长图”“固定 750x5000”“1:6”等时，必须且只能生成 1 张单图，不分页。
- 社交传播优先 `3:4` 或 `4:5`；知识专栏可用 `1:1`；报告摘要可用 `16:9`；单张长图可用 `9:16`、`3:4 长页视觉` 或按用户指定极长比例。

长图拆分后的屏幕职责：

1. 第 1 张：封面 / 主结论，用大字号短标题建立主题。
2. 第 2-3 张：背景 / 痛点 / 核心数据，解释为什么重要。
3. 第 4-6 张：流程 / 细节 / 对比 / 图表，展示方法或证明。
4. 末屏：总结 / CTA / 来源，回收关键结论或行动。

## 核心流程

1. 分析文本与参考图，提取核心信息、层级关系、数据关系和阅读顺序。
2. 判断图表类型、模块结构、张数、比例和是否需要拆屏。
3. 为每张图或每个单张长图模块规划：核心信息、视觉方向、图表类型、完整可见文字。
4. 写出每张图独立的 prompt；如果是系列图，每张都要重申统一视觉系统。
5. 调用 `image_gen` 或 `image_edit`；如基于历史生成图、品牌资产、产品图或版式参考延展，必须使用 `image_edit` 并传入 `image_reference_url_list`。
6. 生成后检查文字、事实、比例、阅读顺序和系列一致性；最多修改一轮。

## 工具适配

- `image_gen`：仅用于无参考图的新信息图、新图表页或新视觉系统首屏。入参使用 `prompt` 和 `ratio`。
- `image_edit`：用于基于用户上传图、历史生成图、品牌资产、产品图、版式参考或上一屏结果继续延展。入参使用 `image_reference_url_list`、`prompt` 和 `ratio`。
- 多屏长图可以逐张调用；当后一张需要继承前一张的版式、色彩或图形母题时，把上一张成功结果的 CDN URL 传入 `image_reference_url_list`。
- 禁止用搜图代替对用户参考图或历史生成图的理解。
- 不把 URL、文件路径、工具参数、模型名、seed、尺寸字段写进 prompt。`ratio` 单独传入。

## 信息结构规则

信息图先定关系，再定样式。不要把所有内容堆成装饰性海报。

| 信息关系 | 推荐表达 | 红线 |
|---|---|---|
| 单个核心数字 | Hero Number / 巨型 KPI 卡片 | 数字必须最大，单位和解释紧贴数字；没有真实数据时用 `“数据待补充”`。 |
| 占比关系 | 环形图、半环仪表盘、堆叠条 | 没有真实比例时不要伪装成精确统计。 |
| 趋势关系 | 折线图、面积图、时间轴 | 坐标、节点和标签要清晰，不能只画抽象曲线。 |
| 流程关系 | 垂直流程、S 型路径、步骤卡片 | 步骤编号连续，箭头方向清楚，说明短句化。 |
| 对比关系 | 左右分栏、矩阵、优缺点表 | 左右结构要对称或明确对抗，不能混淆推荐结论。 |
| 层级关系 | 树状图、金字塔、同心圆 | 严禁把并列关系画成上下级。 |
| 卖点 / 证明 | 证据卡片、图标标签、来源区 | 无真实资质时只能写占位，不画官方认证。 |

## 常见结构骨架

| 类型 / 场景 | 默认结构 | 重点 |
|---|---|---|
| 统计数据型 | 顶部主标题 / 核心 KPI 卡片组 / 主图表 / 底部结论与来源 | 主数字极大，标签清晰，避免花哨 3D 图表。 |
| 流程 / 时间线 | 顶部主标题 / 垂直或 S 型路径 / 节点编号 / 说明卡片 | 路径连贯，步骤短句化。 |
| 对比 / 优缺点 | 顶部标题 / 左右对半分栏 / 对比维度行 / 底部推荐 | 两侧结构清楚，结论醒目。 |
| 产品说明长图 | 首屏大主视觉 / 核心价值 / 局部细节 / 参数与 CTA | 先总后分，屏与屏之间排版交替。 |
| 品牌视觉板 | Logo 区 / 色彩 / 字体 / 图像风格 / 样机 | 网格化排版，避免声称官方认证。 |
| 方法论图解 | 标题 / 3-5 个模块 / 连接关系 / 总结句 | 抽象概念必须转为卡片、箭头、图标和短句。 |

## 视觉风格转换

抽象风格词必须翻译成可见物理特征：

| 用户抽象词 | 转换为具体视觉特征 |
|---|---|
| 专业 / 权威 | 深蓝、冷灰、纯白；严谨网格；清晰平光；标准黑体或无衬线；少于 3 种主色。 |
| 高端 / 高级 | 克制配色；大留白；单侧柔光；极简无衬线或优雅明朝体；弱装饰。 |
| 活泼 / 亲和 | 高明度低饱和色；圆角卡片；气泡、贴纸、手账元素；温暖全局光。 |
| 大气 / 宏大 | 大尺度背景、广角视角、巨大核心数字或主图形；极强层级对比。 |
| 简约 / 极简 | 浅底、单色或双色、少元素、充足留白、几何无衬线。 |
| 科技 / 未来 | 深色底、发光线条、数据面板、HUD 网格、等宽字体、青紫点缀。 |

## Prompt 写作规则

最终 prompt 是直接传给图片工具的一段中文画面描述，不输出内部分析字段。

- 每张图必须按“总览句 + 分区扫描 + 条件收束”写完整。
- 总览句说明构图方向、画面风格、核心信息，不要写内部产物路由名。
- 分区扫描按从上到下、从左到右、从中心到四周描述。
- 每个可见元素都要写清位置、元素类型、形态、内容、文字样式、状态、关联关系、配色材质和视觉处理。
- 所有可见文字必须逐字写出，并用中文双引号包裹。
- 标题短句化，正文卡片化，避免大段自然段挤进画面。
- 证据与数据要视觉化，例如放大数字、进度条、折线图、环形图、矩阵表。
- 每个元素至少有明确颜色或材质描述。
- 条件收束补充全局配色、字体层级、图表清晰度和质量要求：`所有可见文字必须清晰、拼写正确且易于阅读。`
- prompt 末尾不要写尺寸、URL、文件路径、工具字段、模型名或 seed。

图生图 prompt 额外要求：

- 先说明使用 `image_reference_url_list` 中哪张图作为版式、主体、品牌或风格参考。
- 明确保留项：产品形状、Logo、核心图表结构、品牌色、版式比例或上一屏视觉系统。
- 明确编辑项：新增模块、替换文案、重排图表、延展下一屏、调整背景。
- 至少 70% 内容描述目标图，不复述源图背景、旧文案或偶然细节。

## 文案与文字红线

- 画面文字必须短、完整、可读。标题通常 6-14 个字，卡片说明通常 4-12 个字。
- 禁止使用 `XX`、`某某`、`标题文字`、`正文说明`、`数据内容`、`若干文字`、`相关说明` 等占位表达。
- 若必须占位，只能写会真实渲染的低风险文字，例如 `“数据待补充”`、`“来源待补充”`、`“二维码占位”`。
- 金融、健康、医疗、教育升学、投资收益等高风险领域，不得编造效果承诺、收益率、治愈率、官方认证或监管背书。
- 没有用户授权和真实材料时，不画官方 Logo、证书编号、奖章、排名、星级评分或媒体背书。

## 生成前检查

调用图片工具前检查：

1. 是否正确路由到信息图，而不是电商、品牌应用或平台媒体。
2. 是否确定单张 / 多屏 / 单张长图策略。
3. 每张图是否有明确阅读职责，不重复堆叠。
4. 是否需要 `image_edit`；若基于参考图、历史图、品牌资产、产品图或上一屏延展，是否已传入正确的 `image_reference_url_list`。
5. 所有可见文字是否逐字写出并加中文双引号。
6. 数据、来源、认证、功效、收益、排名是否有来源或被降级为安全占位文字。

## 生成后检查

生成后检查：

- 文字是否清晰、完整、无乱码、无裁切。
- 信息顺序是否成立，用户能否按屏幕顺序读懂。
- 图表是否表达了正确关系，没有把并列、趋势、占比、流程混淆。
- 如果是系列图，主色、字体、网格、图形母题和视觉密度是否一致。
- 如果使用 `image_edit`，参考图是否用对，核心资产或上一屏版式是否被保留。

不达标时最多修改一轮，只修正关键偏差；一轮后仍不理想时交付当前最佳结果并说明需要补充的数据或素材。
