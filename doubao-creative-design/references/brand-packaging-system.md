# 包装系统

## 主 Skill 路由适配

本 ref 只提供包装系统的产物规则。被 `doubao-creative-design` 路由调用时，工具协议、澄清轮次、核心资产记录和 prompt 总规则以主 `SKILL.md` 为准；本文件只补充包装结构、包装家族一致性、平面展开、SKU、陈列和印刷文字门禁。

## 概述

### 产物 Skill 边界

包装系统是独立产物 skill。它围绕同一套包装结构、正面层级、材质工艺和陈列场景展开，不默认生成 Logo 全套、视觉语言页或无关活动物料。若用户只要求包装，优先把主包装做清楚；必要时只补货架、开箱或细节系统增强方案完整度。

## 使用场景

### 意图理解与产物分流

先判断用户要解决的是包装本体、结构验证、平面印刷延展、配套开箱体验、货架识别还是 SKU 系列化。不要把所有包装需求都默认理解成 3D mockup 或场景图。

- 用户点名刀模、展开、平面、印刷、出血、安全区、折线、粘口时，优先生成概念刀模 / 平面展开，而不是 3D 场景。
- 用户点名纹样、图案、包装纸、内衬纸、封套、贴纸、封签、吊牌、贺卡、插卡、说明卡时，优先生成平面延展产物，并说明它与主包装母版的互通维度。
- 用户点名开箱、礼盒、套装、配套、仪式感时，至少包含包装本体和配套装饰；标准方案中优先安排一张配套装饰 / 包装纸 / 贺卡平面稿。
- 用户点名货架、上架、陈列、零售、电商主图时，优先确保主包装正面和产品名可读，再选择货架或场景图。
- 用户点名系列、SKU、口味、香型、材质变体时，优先安排 SKU 变体图或色彩系统图；结构和主视觉母题不能重做。
- 用户没有指定产物类型时，默认采用“主包装母版 + 结构或平面延展 + 配套装饰或纹样 + 场景验证”的组合，减少同质化角度图。

### 包装类型选择

| 品类 | 包装类型 | 视觉重点 |
|---|---|---|
| 零食/食品 | 袋装、枕包、盒装、罐装 | 食欲感、货架层级 |
| 礼品/节日 | 硬质礼盒、抽屉盒、套盒 | 仪式感、开盒体验、高级细节 |
| 美妆/护肤 | 瓶器 + 外盒、罐盒、软管 | 材质、信任感、精致度 |
| 饮品 | 瓶标、罐标、纸盒、提篮 | 货架色彩识别 |
| 咖啡/茶 | 牛皮纸袋、铁罐、小袋盒 | 产地感、手作质感 |
| 保健品 | 瓶装、袋装、纸盒 | 干净、可信、信息分区 |
| 玩具/IP/收藏 | 盲盒、开窗盒、卡包 | 角色、收藏感 |
| 零售/服饰 | 手提袋、吊牌、快递盒 | 携带场景、品牌识别 |

## 核心流程

### 张数策略

包装是核心需求时通常生成 3-4 张；如果只是辅助应用，生成 1-2 张即可。

- 轻量：1-2 张，适合主包装正面、单个瓶标 / 袋型 / 礼盒方向、概念刀模、纹样图、包装纸、贺卡 / 插卡或包装局部优化。
- 标准：3-4 张，适合主包装英雄图 + 三视图 / 结构图 + 概念刀模或纹样图 + 配套装饰 / 陈列 / 开箱场景。
- 完整包装系统：5-6 张，适合主包装、三视图、概念展开、打开结构、纹样与包装纸、贺卡 / 封签 / 贴纸 / 吊牌、SKU 变体或货架场景。

多张包装图必须围绕同一套包装家族展开。若两张图只是角度或背景轻微变化，应合并或改成物理差异更明确的产物。

张数分配要服务意图：结构型任务把刀模、三视图、打开结构放在前面；平面延展任务把纹样、包装纸、卡片、封签和贴纸放在前面；展示型任务才把货架、开箱或场景图放在前面。标准 3-4 张中避免全部都是 3D 包装场景，至少保留 1 张结构性或平面延展产物，除非用户明确只要场景 mockup。

### 通用执行内核

本产物 ref 可独立参考。执行顺序固定为：理解用户意图 -> 确认历史资产和参考图角色 -> 制定产物张数与依赖 -> 为每张图写独立 `design_brief` -> 调用 `image_gen` 或 `image_edit` -> 结果加入图片引用表。被主 skill 路由时，工具字段映射以主 `SKILL.md` 的工具协议为准。

- `image_gen` 只在没有用户参考图、历史生成图或核心锚点，且必须建立第一张核心资产时使用；入参为独立 `prompt` 和 `ratio`。
- `image_edit` 用于所有已有用户图、历史图、Logo / IP / 包装 / 主 KV / 视觉语言锚点的延展；入参为独立 `prompt`、`ratio` 和按顺序传入的 `image` 数组。
- 多参考图必须在 prompt 中按 `[img0]`、`[img1]`、`[img2]` 逐张说明作用，编号与 `image` 入参顺序一致。
- `prompt` 只放完整 `design_brief`，不写比例、工具参数、文件路径、URL、模型名、seed、输出文件名或内部 skill 名称。
- `design_brief` 必须完成三层推演：载体骨架、场景语义、实例填充。复杂画面按物理空间分区扫描，每个可见元素写清位置、类型、形态、内容、文字样式、状态、关系、风格和工艺。
- `image_edit` brief 先写生成目标，再写参考图提取项，最后展开目标画面；至少 70% 内容描述目标图，不复述旧图。旧背景、旧裁切、旧文字、偶然道具和噪声只做内部判断，不写进最终 prompt。
- 所有画面文字必须逐字明确，并用中文双引号包裹；每处文字必须写清字体大小和字体风格。用户提供的品牌名、产品名、人名、活动名必须逐字保留。
- 价格、销量、折扣、功效百分比、排名、评分、认证编号、监管声明等数值或权威信息只能来自用户提供内容或已确认资产。
- 高密度信息物料必须先建立 `visible_text_inventory`；最终 prompt 只能渲染清单内文字。没有正文时，用短标题、短标签、按钮、数字、日期、步骤名和图标卡片表达。
- 不使用 `xx`、`某某`、`品牌名`、`对应色值`、`应用场景说明`、`相关说明`、`规范说明`、`说明文字`、`标签文字`、`若干文字`、`辅助信息`、`短说明`、`自然段正文`、`介绍文案`、`补充信息`、`文字占位` 等会让模型自由补字的表述。
- 批量 / 并发仅限同参考图集合、同 `image` 顺序、同 `ratio`、互不依赖且每张都有独立 `design_brief` 的衍生产物；不要把多个正式产物合并到同一画布。
- 传入工具前自检：prompt 中没有负向提示、风险词、工具字段、路径、模型名、画幅参数；所有中文双引号内容都是最终画面文字；重复结构没有用“等”“若干”“同上”省略。

### 核心策略：一致性与多样性

独立运行时也必须先锁定包装家族核心锚点，再做差异化延展。不要为了多样性破坏包装识别，也不要为了一致性让多张图只像同一包装的轻微换角度。

#### 一致性锁定

- 锁定用户明确提供或历史生成确认的包装结构、正面层级、Logo / IP 位置、品类命名、主色比例、纹样逻辑、材质工艺、产品名和短卖点。
- 主包装正面 / 英雄图是包装家族母版；后续三视图、内部结构、平面展开、配套装饰、SKU 变体和陈列场景都必须从母版延展，不重新建立另一套包装本体。
- 同一轮多图必须复用同一套 `image` 参考图顺序和锁定变量；如果某张图不能传参考图，需要在 prompt 中完整复写锁定变量。
- 所有可见文字必须逐字保留用户提供内容，不自行更换品牌名、产品名、SKU 名、口味 / 香型名、认证或数值信息。

#### 多样性释放

- 多图之间至少在闭合正面、三视图、打开结构、平面展开、细节系统、陈列场景、开箱、SKU 色彩或材质变体中选择 2-3 项形成真实差异。
- 差异必须服务用户意图和包装落地；不要只换背景、只换角度或只换装饰。
- 包装系列图必须看起来属于同一套家族，但每张图承担不同功能：确认主视觉、验证结构、展示材质、展示货架识别或扩展 SKU。

#### 主包装到衍生资产

- 包装本体必须保持结构一致：闭合正面、侧面、背面、打开结构、内托和平面展开应共享同一盒型 / 袋型 / 瓶型、开合方式、折线 / 接缝位置、正面比例、Logo 位置和主要信息层级。
- 三视图必须像同一个包装从不同方向观看；侧面和背面可以延续纹样、色块、材料和信息分区，但不能变成另一种盒型、另一套主视觉或另一种品牌语气。
- 内部结构和开箱图必须从外包装结构推出：盒盖、抽屉、内托、封套、开窗、封签、瓶盖、袋口热封等关系要物理成立，不能只做漂亮但无法闭合的装饰组合。
- 配套装饰必须与包装本体至少在 2 个维度互通：Logo / IP 位置、核心纹样、主辅色比例、材质工艺、字体气质、图形母题、封签形状或开箱顺序。
- 吊牌、卡片、封签、贴纸、包装纸、丝带、内衬纸、说明卡、购物袋等配饰可以改变尺寸、裁切、材质或辅助色，但必须能追溯到主包装母版，不要生成与包装无关的配套物料。
- SKU 变体只改变辅助色、口味 / 香型名、局部材质、标签或小图标；包装本体结构、正面层级、主视觉母题和品牌识别位置保持一致。

### 改写策略

- `image_gen`：没有主包装锚点或用户参考图时，先生成主包装正面 / 英雄图，并把它作为后续 `image` 参考。
- `image_edit`：已有包装、Logo、IP 或视觉语言图时，通过 `image` 传入参考图，只提取包装轮廓、开启逻辑、正面层级、Logo/IP 位置、配色比例、纹样、材质工艺和文字风格。
- 包装衍生图必须先说明目标物理差异，例如闭合正面、三视图、打开结构、平面展开、细节系统、货架陈列、开箱或 SKU 变体。
- 文字要服务包装正面识别和生产落地：品牌名、产品名、口味 / 香型名、短卖点逐字写出并用中文双引号“”包裹；价格、认证、条码、监管信息必须来自用户提供内容。
- 不把参考图的旧背景、旧陈列道具、光线阴影或镜头裁切误当成包装锚点，除非用户明确要求保留这些场景。

### 包装方案链路

包装是重点时，方案应围绕同一套包装系统展开，而不是每张图重新设计一次。

推荐链路：

1. 主包装英雄图：确定母版，包括正面视觉、盒型 / 袋型 / 瓶型、品牌名、产品名、主色、纹样、材质和开合逻辑。
2. 3D 三视图：沿用同一包装本体，展示正面、侧面、背面，保持尺寸关系、折线 / 接缝、信息层级和结构连续。
3. 平面展开 / 概念刀模：把同一包装的正面、侧面、背面、封口、贴纸或内衬展开，保持图形位置、接缝和折线对应。
4. 打开结构 / 内托：从同一包装本体推出盒盖、抽屉、内托、封套、开窗或袋口结构，保持外部视觉和内部承托关系一致。
5. 纹样 / 包装纸系统：从主包装母版提取图形母题，生成可连续铺陈的纹样、包装纸、内衬纸或封套平面图。
6. 配套装饰 / 细节系统：吊牌、封签、贴纸、包装纸、丝带、贺卡、插卡、说明卡、内衬纸使用同一 Logo、纹样、材质工艺或图形母题。
7. 陈列 / 开箱场景：使用同一包装和配饰在货架、礼品桌、手持、开箱或门店中展示。
8. SKU / 材质变体：只变辅助色、口味/香型名、局部材质或装饰，不重做结构和主视觉。

如果用户只要求包装，不必强行生成完整 VI；可以补充少量线上封面、货架陈列或礼品卡，让包装方案更完整。

## 输出形式

### 尺寸与画幅默认值

先区分图片画幅和真实生产尺寸：`ratio` 只决定生成图片画幅，真实包装尺寸必须来自用户、产品规格、真实刀模或印厂资料。没有真实尺寸时，只生成概念包装、概念刀模或结构示意，不编造精确毫米尺寸。

| 产物 | 默认画幅 | 用法 |
|---|---|---|
| 主包装英雄图 | 1:1 或 4:5 | 确认包装本体、正面主视觉和材质质感。 |
| 包装正面版式 | 3:4 或 4:5 | 适合盒面、袋面、标签正面和产品名层级。 |
| 三视图 / 结构展示 | 16:9 或 4:3 | 展示正面、侧面、背面或闭合 / 半开结构。 |
| 概念刀模 / 平面展开 | 16:9 或 4:3 | 展开正面、侧面、背面、封口、粘口、折线和接缝。 |
| 瓶标 / 罐标展开 | 3:1 或 2:1 | 表达环绕标签、接缝、主展示面和背标信息区。 |
| 纹样图 / Repeat tile | 1:1 | 展示可重复单元、母题来源、主辅色比例和边缘衔接。 |
| 包装纸平面设计 | 1:1、4:3 或 3:2 | 展示满版纹样、留白节奏、裁切方向和封口关系。 |
| 贺卡 / 插卡 / 说明卡 | 4:3 或 3:4 | 展示卡面版式、祝福语 / 短说明、品牌识别和材质工艺。 |
| 封签 / 贴纸 / 吊牌套版 | 4:3 | 展示不同配饰的平面版式、模切轮廓、孔位和统一纹样。 |
| 配套装饰系统 | 4:3 或 1:1 | 展示吊牌、封签、贴纸、卡片、包装纸、丝带和内衬纸。 |
| 货架 / 开箱 / 陈列 | 16:9 或 4:5 | 展示包装在真实场景中的识别和使用关系。 |

若用户要求“可生产”“印刷稿”“真实刀模”，但没有提供刀模或印厂规格，必须把结果限定为概念示意。可以表达常见印刷边界：出血区、安全区、裁切线、折线、粘口、封口和开窗区；不要承诺生成可直接印刷的源文件。

### 平面包装产物模式

当用户要求平面、刀版、纹样、包装纸、贺卡、插卡、说明卡、封签、贴纸、吊牌、腰封或瓶标展开时，优先选择以下产物模式。

| 产物 | 视角 | 必须表现 | 禁止项 |
|---|---|---|---|
| 概念刀模 / 平面展开 | 正交俯视 | 正面、侧面、背面、顶底面、粘口、折线、裁切线、开窗、接缝。 | 伪装成真实工程刀模、编造精确尺寸、加入摄影场景。 |
| 纹样图 / Repeat tile | 正交平面 | 重复单元、母题来源、边缘衔接、主辅色比例、可连续铺陈节奏。 | 只做普通背景图、边缘不能衔接、引入无关图形。 |
| 包装纸平面设计 | 正交俯视 | 满版纹样、裁切方向、留白节奏、封口方向、与主包装的色彩和图形互通。 | 做成礼物摄影图、只换背景、不说明重复逻辑。 |
| 贺卡 / 插卡 / 说明卡 | 正视或俯视平面 | 卡面版式、短句文字、品牌识别、边框 / 纹样、纸张和工艺。 | 长段正文、模糊祝福语、无品牌关联。 |
| 封签 / 贴纸 / 吊牌套版 | 正交平面 | 多个小物料的模切轮廓、孔位、封口方向、Logo、短标签和统一纹样。 | 每个配饰重新设计、文字太小、无真实用途。 |
| 腰封 / 瓶标 / 罐标展开 | 正交横向展开 | 主展示面、环绕方向、接缝位置、背标信息区、左右连续图形。 | 忽略曲面环绕、前后信息不连续。 |

### 可选产物

- 主包装正面 / 英雄图。
- 3D 三视图：同一包装的正面、侧面、背面。
- 打开结构 / 内托。
- 概念平面展开 / 刀模式面板。
- 纹样图 / Repeat tile。
- 包装纸平面设计图。
- 贺卡 / 插卡 / 说明卡平面图。
- 封签 / 贴纸 / 吊牌套版。
- 腰封 / 瓶标 / 罐标展开。
- 配套装饰 / 细节系统：吊牌、卡片、封签、贴纸、包装纸、丝带、内衬纸、说明卡、纹样、材质。
- 开箱组合。
- 货架 / 陈列 / 礼品桌场景。
- SKU / 配色 / 材质变体。

## 设计规范

### 结构与信息架构

包装设计必须先确定载体结构，再分配信息层级。不要只把图案贴在一个无结构的盒子或袋子上。

- 正面主展示面：品牌识别、产品名、SKU / 口味 / 香型、主视觉、短卖点、净含量或容量信息区；正面第一眼必须能读出是什么产品。
- 侧面：延续主视觉的色块、纹样或材质，承载卖点、使用方式、品牌故事、口味色带、系列编号或开合提示。
- 背面：承载配料 / 成分、营养 / 功效信息区、使用说明、警示语、生产信息、条码 / 二维码预留区和监管信息区；未由用户提供的法定内容只能做信息区，不写具体数值或编号。
- 顶部 / 底部 / 封口：处理开合逻辑、封签、批号区、易撕口、挂孔、瓶盖、袋口热封或防拆结构；这些结构要和三视图、打开结构、概念刀模互相对应。
- 内部结构：内托、隔板、抽屉、说明卡、内衬纸、封套和产品摆放关系必须从外包装结构推出，不做无法闭合或无法承托的装饰组合。

瓶标、罐标、软管和袋装等曲面 / 柔性载体要说明主展示面、环绕方向、接缝位置和背标信息区。概念刀模要说明折线、裁切线、粘口、封口、开窗和正反面对应关系。

### 印刷与生产边界

- 概念设计可以展示出血、安全区、裁切线、折线、粘口、封口、开窗、烫金、击凸、局部 UV、哑光覆膜、丝印、透明窗和特种纸效果。
- 真实出血值、安全区、色彩模式、纸张克重、专色、刀模线型和印后工艺参数必须来自用户或印厂资料；缺失时不要写成确定生产规格。
- 条码、二维码、认证章、监管声明、执行标准、生产许可证、营养成分、功效百分比、批号和日期只使用用户提供内容；未提供时只能表达预留信息区的版式关系，不生成可误导的真实信息。
- 刀模图默认是概念展开，不是印厂可直接使用的工程刀模；除非用户提供真实刀模文件或明确尺寸，否则不要承诺生产可用。

### 平面设计稿规则

平面产物默认采用正交俯视或正视图，画面像设计稿、刀模板、纹样板或印刷平面文件，不像摄影场景、货架陈列或 3D mockup。

- 概念刀模、瓶标展开、包装纸、纹样图、贺卡、插卡、封签、贴纸和吊牌套版不使用透视角度、桌面道具、复杂阴影或真实拍摄背景。
- 平面图要用清晰边界表达裁切线、折线、粘口、安全区、出血区、开窗区、孔位、模切轮廓、接缝和封口方向。
- 平面产物可以表现纸张、压纹、烫金、局部 UV、丝印、哑光覆膜、半透明硫酸纸、牛皮纸和特种纸质感，但不把质感渲染成空间场景。
- 没有真实尺寸时，只表达比例、相对位置和结构关系；不要写确定毫米数、印厂专色号、真实条码或法规编号。
- 平面设计稿必须保留足够留白和可读层级；卡片、贴纸、封签、吊牌等小物料不要塞入长段文字。

### 品类规范提示

根据品类预留必要信息区，但不编造具体法规文本、认证、编号或功效数据。

| 品类 | 必要信息区 |
|---|---|
| 食品 / 零食 / 饮品 | 配料、过敏原、营养信息、净含量、生产信息、条码或二维码预留区。 |
| 酒类 / 咖啡 / 茶 | 容量、产地 / 风味信息、批次或日期区；酒类 ABV 等数值必须由用户提供。 |
| 美妆 / 护肤 | 成分、使用方法、警示语、容量、批号 / 限期使用提示、背标信息区。 |
| 保健品 / 补剂 | 成分或补充剂信息、建议用量、警示语、声明区、瓶标和外盒背标。 |
| 玩具 / IP / 收藏 | 年龄段、警示语、系列编号、角色信息、开窗和收藏卡位置。 |
| 零售 / 服饰 | 尺码 / 材质 / 洗护信息、吊牌、手提袋、封签、快递盒或环保信息区。 |

### 包装延展手法

- `局部放大`：把 Logo、IP 局部、纹样或图形基因放大成包装正面底纹。
- `阵列重复`：把辅助图形做成袋身、盒身、封签或包装纸的连续图案。
- `材质工艺`：根据品类选择烫金、击凸、局部 UV、哑光覆膜、牛皮纸、透明开窗、金属罐、玻璃瓶或软袋。
- `结构表达`：明确盒盖、抽屉、封套、开窗、手提绳、内托、瓶标、罐盖、袋口热封等真实结构。
- `结构同源`：三视图、打开结构和刀模图必须共享同一包装本体的比例、折线、接缝、封口和开合逻辑。
- `配饰互通`：把包装母版的纹样、封签形状、材质工艺、主辅色比例或字体气质转译到吊牌、贴纸、卡片、包装纸、丝带和内衬纸。
- `货架识别`：包装正面品牌名和产品名必须在陈列场景中可读，不能被道具或光效遮挡。

### 纹样与包装纸系统

纹样和包装纸不是装饰背景，必须服务包装纸、内衬纸、封套、贴纸、封签、腰封、卡片或盒身延展。

- 纹样来源必须追溯主包装母版：Logo 局部、IP 元素、产品形态、品类符号、核心图形母题、包装结构线或主视觉局部。
- 纹样类型要明确选择：直排、半落、镜像、散点、边框、角花、中心徽章、满版底纹、腰封连续图案或局部放大图形。
- 纹样层级要明确：大母题用于视觉识别，中母题用于节奏，小底纹用于填充；留白区用于卡片文字、封签、产品名或折线避让。
- 包装纸平面设计必须说明重复单元、裁切方向、封口方向、边缘衔接、主辅色比例和可被配饰复用的局部图形。
- 同一套包装中的纹样、包装纸、贺卡和封签要分工不同：纹样做连续识别，包装纸做包裹体验，贺卡 / 插卡承载短句或信息，封签 / 贴纸承担开合和识别。

### Prompt 策略

包装衍生图必须优先引用主包装图。只提取：

- 包装轮廓；
- 开启逻辑；
- 正面层级；
- Logo/IP 位置；
- 配色比例；
- 纹样；
- 材质工艺；
- 文字风格。
- 本体结构关系：盒型 / 袋型 / 瓶型、折线、接缝、开合方式、内托和封口逻辑。
- 配饰互通维度：哪些纹样、材质、色彩比例、字体或图形母题从主包装延展到吊牌、封签、贴纸、卡片、包装纸、丝带或内衬纸。

不要过度保留参考图的背景、镜头裁切、旧陈列场景、光线、阴影或道具，除非它们是包装识别锚点。

每张包装 prompt 必须说清物理差异：闭合正面、三视图、打开结构、平面展开、概念刀模、纹样图、包装纸、贺卡 / 插卡、封签 / 贴纸 / 吊牌套版、腰封 / 瓶标展开、陈列场景、开箱或变体系列。
在场景示意图 / 场景概念图情况下，必须保持模特、场景、产品包装的和谐度、统一性和氛围感。模特造型、动作、服装、道具、光线、材质和背景都应服务同一品牌气质与使用情境；包装主体保持清晰可见，不被模特、道具、光效或裁切弱化。

### 平面产物 Prompt 模板

以下模板只用于起草 `design_brief`，传入工具前必须替换成具体画面描述；不要把模板标题、字段名、说明语或未提供信息渲染进画面。

#### 概念刀模 / 平面展开

```text
生成一张包装概念刀模平面图，正交俯视，无 3D 透视，无摄影场景。画面展示同一包装本体的正面、侧面、背面、顶部、底部、粘口、封口、折线、裁切线、安全区、出血区和开窗区。正面区域沿用主包装母版的品牌识别、产品名、主视觉、主色比例和纹样，侧面和背面延续同一图形节奏并预留信息区。所有线型清楚区分，结构关系物理成立。没有真实尺寸时只表达比例关系，不写确定毫米数。
```

#### 纹样图 / Repeat tile

```text
生成一张包装纹样平面图，正交平面，无场景。纹样从主包装母版提取核心图形母题、主辅色比例和材质气质，形成可连续铺陈的重复单元。画面说明重复方式是直排、半落、镜像、散点、边框、角花、中心徽章或满版底纹中的具体一种；边缘衔接自然，留白区可用于封签、卡片或包装纸裁切。纹样不能变成普通背景图，必须能用于包装纸、内衬纸、贴纸或盒身延展。
```

#### 包装纸平面设计

```text
生成一张包装纸平面设计图，正交俯视，无礼物摄影场景。画面展示满版包装纸纹样、重复单元、裁切方向、封口方向、边缘衔接、主辅色比例和局部留白。包装纸图形继承主包装母版的 Logo / IP 局部、纹样、字体气质、材质工艺或图形母题，局部图形可以被封签、贴纸、贺卡或内衬纸复用。
```

#### 贺卡 / 插卡 / 说明卡

```text
生成一张贺卡、插卡或说明卡平面设计图，正视或俯视平面，无 3D 场景。画面包含卡片外轮廓、边框、主纹样、品牌识别、短句文字和材质工艺；文字必须来自用户提供内容或本轮已确认内容，并逐字写出。卡片与主包装母版至少在纹样、主辅色、字体气质、材质工艺或图形母题中复用两项，不写长段正文。
```

#### 封签 / 贴纸 / 吊牌套版

```text
生成一张封签、贴纸、吊牌套版平面图，正交平面，无摄影场景。画面展示多个小物料的模切轮廓、孔位、封口方向、短标签、品牌识别、主纹样和材质工艺。每个小物料用途不同：封签用于开合识别，贴纸用于装饰和信息提示，吊牌用于品牌与产品信息。所有配饰追溯主包装母版，不重新设计另一套视觉。
```

### 包装 QA

交付前按以下项检查；不通过时先改 `design_brief` 或重新生成。

- 包装本体一致：主包装、三视图、打开结构、平面展开、开箱和 SKU 变体共享同一结构、开合逻辑、Logo 位置、主视觉母题和信息层级。
- 结构可成立：盒盖、抽屉、内托、封套、开窗、瓶盖、袋口热封、折线、接缝和粘口关系不矛盾。
- 信息准确：品牌名、产品名、SKU 名、口味 / 香型、净含量、价格、认证、条码、监管声明和数值信息都来自用户或已确认资产。
- 文字可读：主展示面品牌名和产品名可读，背标小字不强行生成长段正文，高密度信息只做清晰短标签和信息区。
- 配饰互通：吊牌、封签、贴纸、卡片、包装纸、丝带、内衬纸和购物袋至少复用主包装的纹样、主辅色、材质工艺、字体气质或图形母题中的 2 项。
- SKU 变体克制：只变辅助色、口味 / 香型名、局部材质、标签或小图标，不重做包装结构和正面层级。
- 场景识别：货架、陈列和开箱图中包装正面不被道具遮挡，品牌和产品识别在缩略图中仍成立。
- 去同质化：多张图不能全部是同一包装换角度的 3D 场景；标准 3-4 张中至少有 1 张结构性或平面延展产物，完整系统中至少覆盖结构、纹样 / 包装纸、配饰和场景中的 3 类。
- 平面产物有效：纹样、包装纸、贺卡、插卡、说明卡、封签、贴纸和吊牌不是缩小版主包装贴图，而是有独立版式、真实用途和与主包装互通的视觉维度。
- 生产边界清楚：没有把概念刀模说成真实工程刀模，没有编造出血尺寸、印厂规格、真实条码、认证章、执行标准或法律印章。
