# 应用系统

## 主 Skill 路由适配

本 ref 只提供品牌应用物料的产物规则。被 `doubao-creative-design` 路由调用时，工具协议、澄清轮次、核心资产记录和 prompt 总规则以主 `SKILL.md` 为准；本文件只补充应用物料的张数、载体、版式、文字门禁和一致性要求。

## 产物 Skill 边界

应用系统是独立产物 skill。它基于已有或本轮确认的 Logo、IP、包装、视觉语言、主 KV 或用户参考图，生成真实线上线下触点，不重新发明品牌身份。用户点名宣传册、三折页、落地页、书签、玻璃贴纸、门店效果图或某个具体物料时，第一产物必须就是该物料。

## 张数策略

应用是核心需求时通常生成 2-4 张。应用图用于验证视觉系统在真实触点中的表现，不应改变品牌识别。

- 单一物料：1-2 张，适合单张海报、单个落地页、单张书签、单个门店效果图或单个宣传册封面 / 跨页。
- 标准物料套装：2-4 张，适合宣传册 + 海报 + 立牌、落地页 + 内容卡片 + Banner、门店贴纸 + 实景效果图等聚焦套装。
- 复杂应用系统：5-6 张，适合线上传播 + 线下门店 + 活动物料组合；仍围绕用户点名产物分配主要张数。

用户指定多个不同尺寸时，按尺寸和用途分别生成；可以一次工具调用传入多个独立 prompt，但不要合并到同一张画布。只有同参考图集合、同顺序、同 ratio 且彼此没有依赖时，才可并发。

## 通用执行内核

本产物 ref 可脱离主 MD 独立参考。执行顺序固定为：理解用户意图 -> 确认历史资产和参考图角色 -> 制定产物张数与依赖 -> 为每张图写独立 `design_brief` -> 调用 `image_gen` 或 `image_edit` -> 结果加入图片引用表。被主 skill 路由时，工具字段映射以主 `SKILL.md` 的工具协议为准。

- `image_gen` 只在没有用户参考图、历史生成图或核心锚点，且必须建立第一张核心资产时使用；入参为独立 `prompt` 和 `ratio`。
- `image_edit` 用于所有已有用户图、历史图、Logo / IP / 包装 / 主 KV / 视觉语言锚点的延展；入参为独立 `prompt`、`ratio` 和按顺序传入的 `image` 数组。
- 多参考图必须在 prompt 中按 `[img0]`、`[img1]`、`[img2]` 逐张说明作用，编号与 `image` 入参顺序一致。
- `prompt` 只放完整 `design_brief`，不写比例、工具参数、文件路径、URL、模型名、seed、输出文件名或内部 skill 名称。
- `design_brief` 必须完成三层推演：载体骨架、场景语义、实例填充。复杂画面按物理空间分区扫描，每个可见元素写清位置、类型、形态、内容、文字样式、状态、关系、风格和工艺。
- `image_edit` brief 先写生成目标，再写参考图提取项，最后展开目标画面；至少 70% 内容描述目标图，不复述旧图。旧背景、旧裁切、旧文字、偶然道具和噪声只做内部判断，不写进最终 prompt。
- 所有画面文字必须逐字明确，并用中文双引号包裹；每处文字必须写清字体大小和字体风格。用户提供的品牌名、产品名、人名、活动名必须逐字保留。
- 价格、销量、折扣、功效百分比、排名、评分、认证编号、监管声明等数值或权威信息只能来自用户提供内容或已确认资产。
- 高密度信息物料必须先建立 `visible_text_inventory`；最终 prompt 只能渲染清单内文字。没有正文时，用短标题、短标签、按钮、数字、日期、步骤名和图标卡片表达。
- 不使用 `xx`、`某某`、`品牌名`、`对应色值`、`应用场景说明`、`相关说明`、`规范说明`、`说明文字`、`标签文字`、`若干文字`、`辅助信息`、`短说明`、`自然段正文`、`介绍文案`、`补充信息`、`文字占位` 等会让模型自由补字的表述。
- 批量 / 并发仅限同参考图集合、同 `image` 顺序、同 `ratio`、互不依赖且每张都有独立 `design_brief` 的衍生产物；不要把多个正式产物合并到同一画布。
- 传入工具前自检：prompt 中没有负向提示、风险词、工具字段、路径、模型名、画幅参数；所有中文双引号内容都是最终画面文字；重复结构没有用“等”“若干”“同上”省略。

## 核心策略：一致性与多样性

独立运行时也必须先锁定品牌应用锚点，再做差异化触点延展。不要为了多样性破坏品牌识别，也不要为了一致性让多张应用图只像同一模板换标题。

### 一致性锁定

- 锁定用户明确提供或历史生成确认的品牌名、Logo / IP / 包装 / 主 KV、主色、字体气质、图形母题、材质工艺、画面文字和传播语气。
- 同一轮多图必须复用同一套 `image` 参考图顺序和锁定变量；如果某张图不能传参考图，需要在 prompt 中完整复写锁定变量。
- 所有可见文字必须逐字保留用户提供内容，不自行更换品牌名、活动名、产品名、口号、价格、日期、折扣或数值信息。

### 多样性释放

- 多图之间至少在物料载体、构图、背景色、媒介结构、信息层级、场景、道具、光影或互动路径中选择 2-3 项形成真实差异。
- 差异必须服务用户点名产物和实际用途；不要只换背景、只换角度或只换装饰。
- 应用系列图必须保持同一品牌锚点，但按媒介释放差异：落地页强调屏幕分区和 CTA，宣传册强调封面 / 跨页 / 模块，三折页强调六面折线，门店 / 玻璃贴强调安装区域和空间透视，书签 / 小卡强调实物工艺。

## 改写策略

- `image_gen`：没有任何可用品牌锚点、但必须建立第一张主 KV 或应用主方向时使用。
- `image_edit`：已有用户参考图、历史 Logo、IP、包装、主 KV、门店实拍或空间图时，通过 `image` 传入参考图；品牌应用任务中除建立初步核心资产外，剩余正式产物默认使用 `image_edit`。
- 应用 prompt 必须先说明目标载体和使用场景，再说明从各参考图提取哪些品牌锚点，最后按物理空间顺序描述目标画面。
- 高密度信息物料必须先建立 `visible_text_inventory`。宣传册、三折页、落地页、内容卡片、品牌手册页、活动海报等不能写自然段填充、泛化说明或模糊小字。
- 每个待渲染文字都必须逐字明确，用中文双引号包裹，并写清字体大小和字体风格。没有用户提供正文时，用短标题、短标签、按钮、步骤名、日期、数字和图标卡片承载信息。
- 应用体裁要有差异：传播封面更吸睛，内容卡片更清晰，宣传册更像册页，三折页有折痕和六面结构，落地页像手机页面，立牌重视远距离可读，门店图重视空间透视和安装关系。

## 线上传播应用

线上应用包括平台传播封面 / 首图、内容卡片、轮播页、活动海报、Banner、官网首屏、移动端落地页和活动报名页。线上应用的目标是让用户在手机端或网页首屏快速理解主题，不是展示完整品牌手册。

### 线上结构规则

平台传播封面和内容卡片需要：

- 手机端标题可读；
- 一个明确主视觉；
- 信息密度受控；
- 钩子清晰；
- 符合平台比例。

线上应用默认按以下结构判断，不把同一套版式机械套到所有产物上：

| 产物 | 目标 | 结构重点 |
|---|---|---|
| 平台传播封面 / 首图 | 吸引点击、快速理解主题 | 标题最大，主视觉占比高，2-3 个短卖点即可。 |
| 内容卡片 / 轮播页 | 解释一个重点 | 每张只讲一个核心信息，使用主标题、主视觉、3 个信息点和短 CTA。 |
| 活动海报 / Banner | 强化活动记忆 | 活动主标题、主视觉、时间 / 地点 / CTA 分层清晰。 |
| 官网首屏 | 建立网页第一屏识别 | 顶部导航、Hero 主标题、主视觉、CTA、下方内容露出。 |
| 移动端落地页 | 转化和行动引导 | Hero、价值卡片、流程模块、证明模块、底部 CTA。 |
| 活动报名页 | 说明活动和参与路径 | 活动标题、时间地点、参与方式、亮点、报名入口。 |

相似线上卡片可以做总览图，但正式产物仍应保持独立，除非用户明确要合并展示。

### 营销参考读取策略
当应用产物属于平台传播封面、种草图、活动海报、内容卡片、Banner、首图、头图或平台传播图时，不读取外部 marketing 文件，直接按以下平台传播知识生成画面策略。

#### 通用传播画面规则

| 维度 | 具体规则 |
|---|---|
| 赛道判断 | 美妆、食品、生活方式偏真实场景、低饱和、轻装饰；科技、工具、SaaS 偏清晰结构、几何色块、功能标签；活动、促销、新品偏强标题、强对比、明确 CTA。 |
| 信息密度 | 封面 / 首图控制 3-6 处文字；内容卡片控制 5-8 处文字；活动海报 / Banner 控制 5-10 处文字。不要把完整文案塞进画面。 |
| 字体层级 | 主标题最大，只保留 1 个；副标题比主标题小一档；卖点标签保持同字号；时间、地点、CTA 放在稳定信息区。 |
| 色彩策略 | 优先沿用品牌主色；传播图可增加 1 个高对比辅助色；生活方式图降低饱和度；活动海报可以提高明度、对比和光效。 |
| 装饰元素 | 可使用便签、贴纸、胶囊标签、箭头、手写线条、编号、光效、纹样，但装饰不能压过标题和主视觉。 |
| 合规边界 | 不写绝对化承诺，不伪造平台 UI，不加入平台名称、账号入口、同款、引流字样；没有时间、地点、价格时不要编造。 |

尺寸用于生成画面时的默认建议；用户明确指定平台、比例或画幅时，以用户指定为准。

| 场景 | 建议比例 | 画面策略 |
|---|---|---|
| 小红书 / 种草图 / 生活方式首图 | 3:4 或 4:5 | 使用真实生活场域，主体偏居中或偏上；画面可采用四区网格阅读流：标题区、主视觉区、卖点区、氛围装饰区。标题要强钩子、短句化；卖点用便签、贴纸、手账拼贴承载；整体低饱和、柔和滤镜，保证手机端第一眼可读。 |
| 抖音 / 短视频封面 / 直播预热图 | 9:16 | 主体占比大，标题强冲击，使用高对比色块或强光效；文字短、粗、直接，适合快速扫读。构图要有动势，可使用斜切、放射、速度线、舞台光、近景人物或产品特写。 |
| 快手 / 直播间封面 / 下沉传播图 | 9:16 或 3:4 | 标题更口语化，利益点更直接；人物或产品靠前，暖色、高饱和、强对比更适合；信息少但醒目，CTA 要明确，例如“今晚开抢”“限时福利”。 |
| B 站 / 视频封面 / 知识内容卡 | 16:9 | 标题与人物、产品、图标形成左右或前后关系；可使用知识点标签、描边标题、强轮廓、年轻化配色。重点是让封面像可点击的视频主题，而不是普通海报。 |
| 朋友圈 / 私域传播图 | 1:1 或 3:4 | 画面克制、真实、可信；减少硬广感，避免过度促销。适合真实场景、轻标题、少量卖点和柔和 CTA；语气像朋友分享，不像平台广告。 |
| 公众号封面 / 文章头图 | 横版长图，约 2.35:1；次图 1:1 | 标题可读性优先，版式稳重；主视觉和标题要形成清楚焦点。适合栏目感、摘要式信息层级、横版头图或长图首屏；避免过多贴纸和碎片化装饰。 |
| 活动海报 / Banner | 海报 3:4 或 9:16；Banner 16:9 或 3:1 | 主标题最大，主视觉形成记忆点；时间、地点、CTA 放在底部或侧边稳定信息区。可以加强光效、透视、舞台感和商业广告感，但不能牺牲文字可读性。 |
| 内容卡片 / 轮播页 | 3:4 或 1:1 | 每张只讲一个重点。结构建议为栏目标签、主标题、主视觉、3 个信息点、底部 CTA。信息点使用同一容器和同一字号，避免一张卡里出现多个主题。 |

读取这些策略时，只把它们转译为画面结构和传播语气，不要把“小红书、抖音、快手、B 站、朋友圈、公众号”等平台名称写进最终图片。平台词只能用于内部判断画幅、信息密度、阅读场景和视觉风格。
正确写法：

- `竖版传播封面，顶部强钩子创意字体标题“xxx”，中部品牌主视觉，周围 3 个手写体便签式卖点“xxx”，底部短 CTA黑体“xxx”。`
- `生活方式传播图，真实场域背景，主体居中偏上，低饱和滤镜，手写感标签承载短句“xxx”。`

错误写法：

- `把平台名称或平台用途标签写进画面标题。`
- `在画面右上角添加平台同款、账号入口或站外引流字样。`

### 线上落地页 / 活动页

落地页是品牌转化页面视觉稿，不是普通海报。先判断用户要官网首屏、移动端长页还是活动报名页。

| 类型 | 默认比例 | 结构重点 |
|---|---|---|
| 官网首屏 | 16:9 或 2:1 | 顶部导航、Hero 主标题、主视觉、CTA、首屏下方内容露出。 |
| 移动端落地页 | 9:16 或 3:4 长页视觉 | Hero、价值模块、流程模块、证明模块、底部 CTA。 |
| 活动页 | 3:4 或 9:16 | 活动标题、时间地点、参与方式、亮点、报名入口。 |

落地页内容模块顺序：

1. **Hero 首屏**：品牌 Logo / 导航、主标题、副标题、主 CTA、主视觉图。
2. **价值说明**：3 个核心价值或卖点，用图标卡片或信息块呈现。
3. **流程 / 参与方式**：用 3-4 步说明用户如何参与、购买、预约或报名。
4. **证明模块**：案例、数据、用户反馈、资质或合作方。
5. **底部转化**：重复 CTA，附联系方式或门店 / 官网信息。

落地页 prompt 要写清屏幕边界和模块层级，例如：`画面为移动端品牌落地页长图视觉稿，顶部是 Hero 首屏，向下依次是价值卡片、参与流程、证明模块和底部 CTA；所有模块沿同一栅格对齐，品牌色和字体系统一致。`

移动端落地页默认属于高密度信息场景。用户没有提供完整正文时，把信息压缩成短标题、标签、按钮、数字、日期、步骤和图标卡片。移动端单张长页建议控制在 8-14 处可见文字；如果需要更长叙事，拆成多张页面或轮播页。

移动端落地页文字库存建议：

| 文字类型 | 数量 | 写法 |
|---|---:|---|
| 品牌名 / Logo 文本 | 0-1 | 中号现代无衬线，逐字保留品牌名。 |
| Hero 主标题 | 1 | 超大号粗体无衬线，8-12 个汉字为宜。 |
| 副标题 | 0-1 | 中号现代无衬线，12-18 个汉字为宜。 |
| CTA | 1-2 | 大号粗体无衬线按钮文字，4-8 个汉字。 |
| 模块标题 | 3-5 | 大号粗体无衬线短标题，每处 4-8 个汉字。 |
| 卡片标签 / 步骤名 | 3-6 | 中号现代无衬线短标签，每处 4-10 个汉字。 |
| 数字 / 日期 / 金额 | 0-4 | 等宽数字或粗体数字，必须逐字写出。 |

### 线上传播物料 Prompt 模板

线上传播物料最容易因为文字过多、信息层级不清和平台 UI 干扰导致文字不可读。撰写 prompt 前先内部确认 3 件事：主标题、主视觉、最多 3 个卖点。正式 prompt 中不要保留 `{变量}`，必须替换为真实画面文字，且所有要渲染的文字都用中文双引号包裹。

通用文字清晰度约束：

- 平台传播封面：画面文字控制在 3-6 处，主标题最大，副标题和卖点必须短。
- 内容卡片：每张只讲 1 个重点，画面文字控制在 5-8 处。
- 活动海报 / Banner：文字控制在 5-10 处，必须明确主标题、时间、地点或 CTA；没有信息时不要编造。
- 移动端落地页：画面文字控制在 8-14 处，文字都放进 Hero、卡片、按钮、流程条、进度环或底部 CTA 等明确容器。
- 每一处文字都要写清字体大小和字体风格，例如大号粗体无衬线主标题、中号现代无衬线副标题、中号几何无衬线卖点标签、较小但清晰的无衬线辅助信息。

## 物料延展原则

应用物料不是把 Logo 贴到不同 mockup 上。每个物料都需要把核心视觉基因转成该载体自己的设计语言。

常用延展方式：

- `局部放大`：截取 Logo、IP、包装纹样或辅助图形中最有识别度的局部，放大成封面、手提袋、海报或店面墙面的超级符号。
- `阵列重复`：把图形标、纹样、角色局部、品牌符号做成规律平铺，用于信封内衬、胶带、包装纸、地贴和社交背景。
- `材质碰撞`：根据载体选择烫金、击凸、丝网印刷、哑光覆膜、亚克力、金属徽章、帆布印花等工艺。
- `结构适配`：不同载体按真实物理结构设计，例如立牌考虑支架和观看距离，门店考虑灯箱和动线，传播封面考虑手机端首屏。
- `场景化陈列`：包装、周边和活动物料需要在货架、礼品桌、展位、门店或用户使用场景中验证识别度。

如果画面只是“居中 Logo + 空背景”，通常不合格；除非用户明确要求极简规范页或单 Logo 展示。

## 体裁调性规范

不同应用体裁应共享同一套品牌锚点，但画面目标和表达强度需要不同。不要把同一套版式机械套到所有物料上。

| 体裁 | 主要目标 | 调性与画面要求 |
|---|---|---|
| 平台传播封面 / 首图 | 吸引点击、快速理解主题 | 标题更大、更有钩子；主体占比高；色彩对比和情绪更强；信息少而清晰；适合手机端第一眼识别。 |
| 内容卡片 / 轮播页 | 解释卖点、承接阅读 | 信息层级清楚；每页只讲一个重点；图文比例均衡；比封面更理性，但仍要保持轻传播感。 |
| 宣传海报 / 活动海报 | 形成营销记忆、传达核心信息 | 主标题、利益点、时间地点或 CTA 更突出；构图更有冲击力；允许更强光效、动势和商业广告感。 |
| 主 KV | 建立统一视觉源 | 品牌符号、主题概念和主画面最完整；不塞太多细节；后续物料都从这里继承色彩、图形和字体气质。 |
| 线下立牌 / 易拉宝 | 远距离识别和现场引导 | 文字更少更大；主体垂直居中；保留安全边距；需要体现材质、支架和真实尺度。 |
| 导视 / 指示牌 | 快速找到方向 | 箭头、区域名和路径优先；装饰克制；对比度高；不要为了美观牺牲可读性。 |
| 门店 / 展位 / 展陈 | 验证空间落地 | 需要真实空间尺度、材质、灯光、动线和安装关系；品牌元素应融入墙面、柜台、灯箱、地贴等触点。 |
| 包装陈列 / 货架场景 | 促进购买和展示系列感 | 包装正面识别必须清楚；重复陈列形成货架冲击；可加入道具，但不能遮挡主包装。 |
| 贴纸包 / 表情包 | 高频传播和互动 | 单个贴纸轮廓清楚；动作和表情差异明显；文字短；适合透明底或白描边。 |
| 周边组合 | 展示可携带触点 | 物料之间要有材质差异，如帆布、亚克力、金属、纸卡；图形可简化，但必须继承核心识别。 |
| 办公物料 / 名片信纸 | 建立专业触点 | 克制、清晰、留白足；突出 Logo、姓名、联系信息和纸张材质；不做过度营销化。 |
| 视觉规范页 | 解释系统规则 | 更像手册页面；分区明确、标签短、样本整齐；避免广告化和过度场景化。 |

Prompt 中要明确当前体裁的目标。例如传播封面写“吸引点击、标题强钩子、手机端可读”，宣传海报写“营销主张、主标题、活动信息和 CTA”，立牌写“远距离阅读、真实支架、垂直版式”。不要让最终画面出现平台名称或“某平台封面”这类内部用途词。

## 线下 / 实体应用

立牌、门店、展位、陈列需要：

- 说明观看距离；
- 展示人体尺度或物理尺度；
- 写清材质和安装环境；
- 描述灯光和空间关系；
- 远距离需要读到的文字必须足够大。

用户要求空间场景时，不要只做一张平面海报。

实体印刷物料要区分平面稿和效果图：

- 名片、信纸、包装展开、导视牌面：优先描述正交平面视角，版面边界清楚，不做夸张透视。
- 包装、手提袋、立牌、展位、门店：优先描述真实 mockup、材质、光源、阴影、尺度和环境。
- 同一物料需要同时展示生产稿和效果图时，拆成两张；只有总览说明页才可以并排放在一张图中。

## 空间贴纸 / 墙纸 / 窗膜规则

当用户要求门店玻璃贴纸、墙纸、装修贴膜、窗膜、门头贴纸或空间局部贴纸时，任务属于空间应用，不默认先生成配色规范页或完整 VI 核心页。

优先产物：

- **贴纸 / 墙纸设计稿**：独立平面稿，展示图案、纹样、色块、透明度和边界形状。
- **实景应用效果图**：基于用户提供的门店、玻璃、墙面或柜台实景图进行 `image_edit`。
- **施工尺寸稿**：只有用户明确要求施工、尺寸、打样或矢量稿时再生成。

参考图使用：

- 门店实景图提供空间结构、透视、光线、玻璃 / 墙面 / 柜台位置。
- 配色图或风格图只提供主色、辅助色、材质和纹样方向，不作为第一产物。
- 多张门店视角应分别作为不同 `image_edit` 任务处理；不同视角比例不同，就分别保留各自比例。

空间效果图必须保留原始空间的主要透视、采光、硬装结构和家具关系，只在玻璃、墙面、贴纸、窗膜、门头、腰线或局部装饰区域做视觉延展。

贴纸 / 窗膜设计应有真实施工逻辑：

- 半透明膜、磨砂膜、渐变膜、腰线贴、门头识别、导视箭头、局部品牌纹样、边角装饰、橱窗展示区；
- 贴纸边缘、透光关系、反射、高度和视线遮挡关系清楚；
- 图案遵循门窗分割、墙面转角、柜台边缘和人体视线高度。

空间贴纸 prompt 示例：

```text
参考 [img0] 的门店入口实景透视和自然光，参考 [img1] 的浅木色、米白、柔和绿色和暖橙色比例，生成门店入口玻璃贴纸应用效果图。画面保留原始门店天花、地面、家具、窗框和采光结构；玻璃区域添加半透明磨砂窗膜、腰线式品牌纹样和局部门头识别，贴纸沿玻璃分割线展开，边缘与窗框对齐，室内仍能透出柔和光线。
```

## 小型印刷物料规则

书签、信封、小卡、邀请卡、优惠券、副券和礼赠卡都需要真实物理比例，不能做成普通海报或随意方卡。

| 物料 | 默认比例 / 结构 | 设计重点 |
|---|---|---|
| 书签 | 竖版细长，约 1:3 到 1:4，例如 50×150mm / 50×180mm | 顶部圆角或打孔区，中部主图形 / 品牌纹样，底部品牌名或短句。 |
| 撕券式书签 | 竖版细长 + 底部可撕副券 | 模切线、撕线、副券小图案和主书签保持一体化。 |
| 信封 | 横向矩形，含封舌、开合线、压纹或烫金 | 信封正面克制，品牌图形可做局部压纹或边角纹样。 |
| 小卡 | 名片比例、3:4 或 A6 近似比例 | 正面主视觉，背面信息或活动说明，适合与信封组合。 |

书签设计要强调可拿在手里的真实工艺：纸张克重、圆角、打孔、丝带孔、模切边缘、压纹、烫金、局部 UV、可撕副券、细窄留白。装饰必须服务于品牌识别和生产可行性。

信封 + 小卡组合图应使用统一光源和真实桌面平铺。信封、小卡、书签各自保持物理尺寸关系；书签保持细长竖版，不被拉宽成卡片或海报。

书签 prompt 示例：

```text
生成一枚品牌书签平面设计稿，画面为竖版细长比例，顶部为圆角和小打孔区，中部是品牌纹样与主图形纵向排布，底部有较小但清晰的品牌短句“{短句}”。纸张表现为厚卡纸质感，边缘有细腻模切，局部图形使用淡金色烫印，整体轻巧、克制、适合线下活动赠品。
```

## 活动系统规则

活动类优先考虑：

- 主 KV；
- 签到 / 拍照背景；
- 导视 / 指示牌；
- 议程板 / 易拉宝；
- 展位 / 陈列场景；
- 传播封面；
- 工作证 / 手册；
- 周边 / 贴纸。

用户提供活动名、日期、地点和 5W1H 信息时，必须保留。

## 宣传册 / 折页规则

这些产物属于应用物料。用户明确指定其中一种时，优先生成该物料本身，不默认扩展为完整 VI、品牌规范页或多渠道全案。落地页 / 活动页归入“线上传播应用”处理。

### 宣传册 / 画册 / 产品手册

宣传册任务要先判断是单页宣传单、代表性跨页，还是完整册页概览：

| 类型 | 默认画面 | 结构重点 |
|---|---|---|
| 单页宣传单 | 竖版 3:4 或 A4 近似比例 | 顶部品牌识别、主标题、主视觉、3-5 个卖点、底部 CTA / 联系方式。 |
| 代表性跨页 | 横版 16:9 或 4:3 | 左页品牌故事 / 场景图，右页产品或服务模块，形成完整阅读动线。 |
| 册页概览 | 多页缩略总览或平铺样机 | 封面、内页、信息页和封底保持同一品牌系统，用于展示系列感。 |

宣传册 / 画册的内容架构：

- **封面**：品牌 Logo、主标题、短副标题、强主视觉。
- **导语 / 品牌故事**：一句核心理念 + 场景图或品牌符号。
- **产品 / 服务模块**：每个模块包含标题、短句标签、图标或图片。
- **优势证明**：数据、资质、用户反馈、流程或对比信息，按品牌可信度选择。
- **行动引导**：电话、网址、门店地址、预约语、咨询按钮等真实可见文字。

单图只生成一张宣传册视觉时，不要假装输出完整多页书；应明确为封面、代表性跨页、单页宣传单或册页概览其中一种。

### 折页 / 三折页

折页是物理印刷结构，必须写清折法、折痕和面板。三折页设计稿本身可以在同一张图片中呈现正反面，这是该物料的结构展示，不属于把无关产物合并到同一画布。

| 折法 | 物理结构 | 单图展示建议 | `ratio` 入参建议 |
|---|---|---|---|
| 对折页 | 1 条居中竖向折痕，2 个等宽面板 | 上方正面、下方反面；正反等宽等高 | 1:2 或 16:9 |
| 三折页 | 2 条等距竖向折痕，3 个等宽连续面板，共 6 个版面 | 上方正面 3 面板，下方反面 3 面板；中间留物理间距 | 3:2 |
| Z 折页 | 2 条等距竖向折痕，3 个等宽连续面板，一正一反交替 | 平面展开稿优先；样机需要呈手风琴式折叠 | 3:2 |
| 四折页 | 3 条等距竖向折痕，4 个等宽连续面板 | 正反同框或单面展开，按用户要求选择 | 2:1 |

三折页正面结构：

- **封面面板**：品牌 Logo、大号主标题、短副标题、Hero Image 或强识别图形。
- **封底面板**：联系方式、地址、网址、预约语或门店信息。
- **折入面板**：品牌简介、服务摘要、活动亮点或次级卖点。

三折页反面结构：

- **核心卖点区**：横跨 2-3 个面板，使用标题、短句标签、图标或局部图。
- **细节展示区**：产品 / 服务 / 场景细节图，配短标签。
- **行动引导区**：预约、咨询、到店、参与或购买路径。

折页 prompt 必须正向写出物理结构，例如：`画面为三折页平面展开设计稿，上方一排是正面 3 个等宽连续面板，下方一排是反面 3 个等宽连续面板，两条等距竖向折痕贯穿上下两排，六个版面属于同一张纸。`

## Prompt 策略

应用类 `image_edit` 时，先说明要生成的新应用图。只从参考图提取可复用锚点：

- 品牌名 / Logo 线索；
- 配色比例；
- 字体气质；
- 母题 / 纹样；
- 材质工艺；
- IP / 包装线索。

不要过度保留参考图的原始版式、背景、裁切、光线、道具和偶然细节。

如果目标是新的应用场景，至少 70% 的 prompt 用于描述目标图：载体、主体、构图、信息层级、尺度、场景、材质和画面文字。

每个应用 prompt 还必须写清该体裁的差异化目标，例如“更吸睛”“更规范”“更适合远距离识别”“更强调购买转化”“更强调空间落地”。这些差异只能改变表达方式，不能改变品牌核心识别。

## 高密度应用文字策略

以下应用产物默认进入高密度信息文字门禁：宣传册、画册、产品手册、招商册、折页、三折页、落地页、活动页、内容卡片、信息图、轮播页、带流程和权益的活动海报、包装背标、说明卡和使用步骤页。

通用处理：

- 先内部列出可见文字库存，再写 prompt；最终画面文字只来自库存。
- 用户没有提供正文时，用短标题、短标签、按钮、数字、日期、步骤和图标承载信息。
- 高密度信息优先拆成卡片、流程条、时间轴、进度环、对比表、图标网格和分栏模块。
- 单图承载过多内容时，优先拆成多张页面、轮播页、册页概览或正反面结构稿。
- 抽象主张要改写为可视化模块。例如品牌理念、服务优势、活动权益和产品卖点，优先拆成主标题、3 个短标签、1 个证明模块和 1 个 CTA。
- 证据模块要有明确视觉载体。讲进度用进度环，讲流程用步骤条，讲材质用微距样本，讲对比用左右对照，讲权益用徽章或票券。
- 数字、日期、价格、折扣、排名、评分、认证和门店数量只使用用户提供内容。用户未提供时，用不含数值的短标签承载信息。
- 多个相似卡片必须逐个写差异：每张卡片有不同图标构成、不同标题、不同色块或不同局部图片。

| 场景 | 文字上限建议 | 结构策略 |
|---|---:|---|
| 移动端落地页 | 8-14 处 | Hero、3 张价值卡、3 步流程、1 个证明模块、底部 CTA。 |
| 内容卡片 / 信息图 | 5-8 处 | 1 个主标题、3 个信息点、1 个 CTA 或结论。 |
| 宣传册单页 | 8-12 处 | 品牌识别、主标题、3-5 个卖点、底部 CTA / 联系方式。 |
| 代表性跨页 | 10-14 处 | 左页故事或场景，右页产品 / 服务模块，文字分栏。 |
| 三折页 | 12-16 处 | 6 个面板各有明确角色，封面、封底、折入页和内页分工清楚。 |
| 活动海报 / 活动页 | 5-10 处 | 主标题、时间、地点、3 个亮点、报名或参与 CTA。 |
| 包装背标 / 说明卡 | 8-12 处 | 规格、步骤、图标、短句提示，正文来自用户提供内容。 |

高密度场景的 prompt 要把“文字”和“容器”绑定描述，例如 `价值卡片标题为中号现代无衬线文字“{标签1}”`、`流程步骤卡内显示中号几何无衬线文字“{步骤1}”`、`进度环中心显示大号等宽数字“{数字1}”`。不要单独堆一段文字清单。

文字改写时优先使用以下结构：

- 主标题：把抽象概念压成一个具体结果或行动，例如把品牌理念改成一句可读主张。
- 卖点标签：每个标签只讲一个利益点，3 个标签之间在内容和长度上形成差异。
- 证明信息：把文字证明改成数字卡、徽章、进度环、对比块或图标阵列；数值必须来自用户。
- 行动指令：按钮或 CTA 使用短动词短语，例如 `立即参与`、`预约咨询`、`查看系列`。
- 图标说明：描述图标的形状和颜色，例如 `由橙色圆环和白色对勾组成的徽章`，不只写功能名。
