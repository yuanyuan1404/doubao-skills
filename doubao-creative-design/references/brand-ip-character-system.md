# 品牌 IP 角色系统

## 主 Skill 路由适配

本 ref 只提供 IP / 角色系统的产物规则。被 `doubao-creative-design` 路由调用时，工具协议、澄清轮次、核心资产记录和 prompt 总规则以主 `SKILL.md` 为准；本文件只补充核心 IP 锚点、角色一致性、资产顺序和角色类 prompt 规则。

根据方案生成以角色为核心的 VI 资产。第一张 IP 图像是身份锚点；后续所有角色图像都必须继承它。

## 默认输出模式

默认先在内部规划结构化任务条目，不只写 `prompt`。核心图任务标记为 `tool_mode: core_gen`，生成后记录 `core-ip` 的 CDN URL；所有衍生资产任务标记为 `tool_mode: derivative_edit`，并必须带 `image_reference_url_list: ["{core-ip CDN URL}"]`。这些字段用于内部规划和一致性检查；实际工具调用时按主 `SKILL.md` 映射为 `image_gen` 或 `image_edit` 的可用字段。

## 必需输入

- 品牌/IP 名称、品类、受众、性格。
- 参考图像，以及需要保留的内容：形态、渲染方式、配色、材质、版式、情绪。
- 锁定的角色变量：物种/身体类型、头身比、脸部、眼睛、嘴/鼻、服装、道具、配色、材质、性格。
- 交付物列表、主题方向、画幅比例、可见文字。

如果尚无角色形象，先生成 `核心视觉资产` 条目，并将其记录为后续衍生图的来源。

## 工具适配

### 文生图工具

用途：只用于生成或备选生成核心 IP 形象 `core-ip`。除核心形象外，不得用于主视觉、三视图、设定卡、海报、表情包、服装版本、主题套图、周边等衍生资产。

入参：

- `prompt`：完整画面描述，必须保留本技能要求的文生图结构。
- `ratio`：工具支持的比例，例如 `1:1`、`3:4`、`4:5`、`16:9`、`4:3`。

出参：

- 生成图片 URL、文件或图片对象。

### 图生图工具 / image_edit

用途：基于用户上传图、历史生成图或核心视觉资产进行 IP 衍生、编辑、风格化、替换、延展和系列化。所有非核心资产默认都必须使用此类工具。

入参：

- `image`：file、URL 或 image object。
- `prompt`：仅描述修改目标、角色锚点提取、目标画面和保留要求。
- 必要时传入工具支持的其他参数。

出参：

- 编辑后的图片 URL、文件或图片对象。

### 比例和尺寸决策

文生图和图生图都按以下顺序决定画布尺寸：

1. 优先遵循用户明确的尺寸意图。
   例如明确写出 `1:1`、`3:4`、`4:5`、`16:9`、`4:3`，或指定主视觉、三视图、表情包、周边图、海报、横幅、设定卡等交付物。
2. 如果用户没有明确尺寸且任务使用参考图，判断是否需要沿用参考图尺寸或比例。
   当任务是基于核心视觉资产做轻编辑、风格保持、同构图延展、表情包延展、主视觉系列化或应用图变体，且构图连续性重要时，优先保持参考图比例。
3. 如果以上都不适用，则按交付物推荐合适比例。
   主视觉 `16:9` 或 `4:5`；角色圣经页 `16:9`；设定卡 `16:9` 或 `4:5`；三视图 `16:9` 或 `4:3`；表情页 `16:9` 或 `4:3`；动作延展 `16:9`；场景海报 `4:5` 或 `16:9`；表情包 `1:1` 或 `4:5`；周边套组 `16:9` 或 `4:5`；IP + Logo 组合 `16:9` 或 `4:3`。

### 调用逻辑
1. 串行执行
- 先生成或确定核心 IP 形象。用户已有核心参考图时，可直接作为 `core-ip` 参考 URL，也可按用户意图调用 edit 生成后将结果 CDN URL 记录为 `core-ip`；用户没有核心图时，先调用 gen 生成 `core-ip` 核心形象。
- 这一步可以不等用户确认，直接进入后续产物，但后续所有产物必须基于同一个 `core-ip` CDN URL。
- 后续图片如果必须基于前一张实际生成结果继续生成，或用户明确要求中途确认，则串行执行。
2. 批量 / 并发执行
- 如果图生图工具支持一次传入多个 `prompt`，优先使用 batch edit：同一批多个 prompt 共享同一个 `image_reference_url_list: ["{core-ip CDN URL}"]`，每个 `prompt` 对应一张独立图片，不能把多个独立资产拼到同一张画布。
- 同批任务的每个条目都必须有独立的 `tool_mode`、`image_reference_url_list`、`asset_id`、`title`、`prompt`、`visible_text` 和 `ratio`，并能按 `asset_id` 映射回对应结果。
- 后续所有衍生资产必须使用 edit / 图生图，并传入同一个 `core-ip` CDN URL；禁止各资产独立调用纯 gen 生成。
- `ip-main-view`、`ip-turnaround`、`ip-setting-card`、`ip-character-bible` 可以同批输出，前提是同批每个任务都基于同一个 `core-ip` CDN URL。
- `ip-scene-poster`、`ip-concept-poster`、`ip-social-scene-cards`、`ip-theme-suite` 可以同批输出，前提是同批每个任务都基于同一个 `core-ip` CDN URL，且主题方向、场景范围和可见文字已锁定。
- 姿态扩展类不默认全量输出，只选 1-2 个最贴合用户意图的类别。`ip-action-extension`、`ip-expression-sheet`、`ip-sticker`、`ip-costume-variation`、`ip-merchandise-set`、`ip-logo-combination` 可以同批输出，前提是同批每个任务都基于同一个 `core-ip` CDN URL，且角色服装、道具、表情风格、Logo 和品牌文字已锁定。
- 各批之间只要不依赖上一批的真实生成结果，就可以并发启动。
3. 多尺寸处理
- 多尺寸或多比例通常需要多次工具调用。若工具只支持单次调用一个 `ratio`，按 `ratio` 分组批量调用；不同 `ratio` 组之间可以并发执行。
- 若工具支持每个 `prompt` 独立指定 `ratio`，允许在同一次批量调用中混合比例。
- 同一资产输出多个比例时，拆成多个独立条目，例如 `ip-main-view-16x9`、`ip-main-view-4x5`；它们可以并发生成，但各自必须保留完整独立提示词。
- 只有资产本身定义为组合型画面时，才在单张画布内组织多个内容模块，例如 `ip-turnaround`、`ip-action-extension`、`ip-costume-variation`、`ip-social-scene-cards`、`ip-concept-process-board`、`ip-system-longform`。不要为了减少调用次数把无关资产硬拼成一张图。

### 工具路由硬规则

| 任务类型 | 必须使用 | 必填字段 |
|---|---|---|
| 核心 IP 形象 | `tool_mode: core_gen` 或用户参考图 edit 后定为 `core-ip` | `asset_id: core-ip`、`prompt`、`ratio`、生成结果 CDN URL |
| 所有衍生资产 | `tool_mode: derivative_edit` | `image_reference_url_list: ["{core-ip CDN URL}"]`、`asset_id`、`prompt`、`visible_text`、`ratio` |

- 文生图只允许生成 `core-ip` 或核心候选；核心图确定后，后续资产必须走 edit / 图生图。
- 如果 `core-ip` CDN URL 未成功返回，必须先补齐核心资产 URL；禁止改用 gen 继续生成衍生资产。
- 如果发现三视图、设定卡、角色海报、概念海报、表情包、服装版本、主题套图、周边等衍生资产的 `tool_mode` 是 gen，必须改为 `derivative_edit` 后再执行。

### 核心 IP 锚点门禁规则

- 任何 IP 方案必须先生成或确定 `core-ip` CDN URL，再生成其他衍生资产。
- 用户已有核心参考图时，可直接作为 `core-ip` 参考 URL；如果用户意图是优化、风格化或改造成 IP，则先用 edit 处理参考图，并将结果 CDN URL 记录为 `core-ip`。
- 用户没有核心图时，先用 gen 生成 `core-ip` 核心形象；无需等待用户确认也可以继续，但后续必须在 `image_reference_url_list` 中传入这个 `core-ip` CDN URL。
- 后续所有资产，包括主视觉、三视图、设定卡、角色海报、概念海报、表情包、服装版本、主题套图、周边等，都必须基于同一个 `core-ip` CDN URL 调用 edit / 图生图。
- 后续 prompt 必须写明：基于 `core-ip` 保持脸部结构、头身比、物种、眼睛、嘴鼻、主服装、主色、材质和核心道具，只变化姿势、表情、视角、场景、局部道具、排版和应用形式。
- 绝对禁止 `ip-main-view`、`ip-turnaround`、`ip-setting-card`、海报、表情包等分别独立文生图；也禁止把多张独立生成图当作同一个 IP。
- 三视图是最高一致性资产，必须基于 `core-ip` CDN URL 生成；不能只靠文字描述“同一个角色”。

### 生成顺序与张数分布策略

1. 角色锚点阶段
- 先确定 `core-ip` CDN URL。用户有核心参考图则直接或 edit 后记录为 `core-ip`；用户没有核心图则先 gen 一张 `core-ip`。
- 提取角色锚点：物种、脸部结构、头身比、眼睛、嘴鼻、主服装、主色、材质、核心道具和性格。
2. 基础设定阶段
- 默认输出 2-3 张，优先选择 `ip-main-view`、`ip-turnaround`、`ip-setting-card` 或 `ip-character-bible`。
- 目标是先固定官方形象、三视图、设定信息和基础视觉语言。
3. 传播场景阶段
- 默认输出 1-3 张，优先选择 `ip-scene-poster`、`ip-concept-poster`、`ip-theme-suite` 或 `ip-social-scene-cards`。
- 目标是生成更吸睛、更丰富、更有动作和场景叙事的传播图。
4. 姿态扩展阶段
- 默认输出 1 张，完整方案最多 2 张；只选择 1-2 个最贴合用户意图的类别，不要把动作、表情、贴纸、服装、周边全部堆满。
- 用户强调穿搭、制服、节日、职业、系列皮肤时，选 `ip-costume-variation`。
- 用户强调动作、玩法、任务、场景互动时，选 `ip-action-extension`。
- 用户强调社交传播、表情包、可爱反应时，选 `ip-expression-sheet` 或 `ip-sticker`。
- 用户强调落地应用、周边、商业化时，选 `ip-merchandise-set`。
- 用户没有明确要求时，标准方案默认选 `ip-action-extension`；完整方案可加 `ip-costume-variation`。
5. 整合展示阶段
- 只有用户要求作品集长图、提案页或总结展示时，才生成 `ip-system-longform`；它不默认占常规张数。

| 方案规模 | 推荐张数 | 分布 |
|---|---:|---|
| 快速方案 | 3 张 | 主视觉 1、三视图/设定 1、场景海报/主题图 1 |
| 标准方案 | 4-5 张 | 基础设定 2、传播场景 1-2、姿态扩展 1 |
| 完整方案 | 6-7 张 | 基础设定 2-3、传播场景 2、姿态扩展 1-2、可选过程板 1 |
| 作品集方案 | 8-10 张 | 基础设定 3、传播场景 3、姿态扩展 2、周边/Logo 1、长图整合 1 |

### 单点编辑与多图输出策略

- 单点编辑任务默认输出 1 张。例如用户只要求“改一下表情”“换个服装”“生成一张海报”“优化这张设定卡”，默认只生成 1 个资产。
- 多图输出只在用户明确要求“套图、整套、完整方案、多张、系列、批量、四宫格、九宫格、三视图、动作展示、周边套组”等意图时触发。
- 用户明确指定张数时，优先按用户张数执行；若张数与资产目标冲突，合并相似资产或简短说明取舍。
- 用户未指定张数但意图是完整 IP 方案时，按方案规模推荐合理张数：快速 3 张、标准 4-5 张、完整 6-7 张、作品集 8-10 张。
- 用户意图只是轻量修改或单资产生成时，不主动扩展为完整 IP 套装。

### 用户意图与模糊度决策策略

先按 prompt 模糊度决定方案规模，再按主要意图选择内容。用户明确指定张数、比例或资产列表时优先服从；若资产重复、冲突或文字缺失，再澄清。

判断四项关键信息：品牌/品类、品牌理念/IP 使命/性格世界观、风格调性、交付类型/用途范围。缺失 3 项及以上时不要逐项追问，**只问一个**对方案影响最大的问题：优先问 IP 理念；理念已明确但用途不清，问交付范围；理念和用途已明确但风格不清，问风格调性。轻微模糊时不提问，内部选择最贴近意图的方案，并只在最终资产清单或 QA 备注中自然交代取舍；完整执行计划只在内部维护，不向用户展示。

| 模糊度 | 判断标准 | 默认策略 |
|---|---|---|
| 高模糊 | 只说做 IP、设计角色、参考风格；四项关键信息缺失 3 项及以上 | 只问 1 个关键问题；若用户要求直接做，用快速方案 3 张 |
| 中模糊 | 有品牌、主题或参考图，但未指定张数/交付物 | 用标准方案 4-5 张 |
| 低模糊 | 品牌、角色方向、用途、参考图、文字或比例较明确 | 用完整方案 6-7 张；若用户给资产列表则按列表执行 |
| 明确指定 | 直接指定张数、比例、资产 ID 或输出顺序 | 按指定执行，并合并相似度超过 70% 的重复资产 |

| 主要意图 | 优先内容 |
|---|---|
| 做一个 IP / 吉祥物 | `ip-main-view`、`ip-turnaround` 或 `ip-setting-card`、`ip-scene-poster` |
| 完整 IP 方案 / VI / 作品集 | 基础设定、传播场景、1-2 个姿态扩展、可选 `ip-system-longform` |
| 角色海报 / 宣传图 | `ip-scene-poster` 优先，强调强动作、强构图、标题压版和丰富场景 |
| 概念海报 / 世界观 | `ip-concept-poster` 优先，强调叙事空间、氛围、探索感和文字排版 |
| 主题活动 / 节日 / 城市 / 科技 | `ip-theme-suite`、`ip-scene-poster`、应用物料或社媒场景卡 |
| 表情包 / 社交传播 | `ip-expression-sheet` 或 `ip-sticker` |
| 穿搭 / 制服 / 多版本 | `ip-costume-variation` |
| 周边 / 落地应用 / 商业化 | `ip-merchandise-set`、`ip-logo-combination`、应用物料 |
| 影视角色 / 剧情角色 / 电影感角色 | `ip-cinematic-character-concept`、`ip-costume-prop-breakdown`、`ip-scene-moment-board`、`ip-production-reference` |

短问题模板：
- 理念问：这个 IP 主要传达什么品牌理念？A. 陪伴信任：温暖、亲和、适合大众传播；B. 探索成长：好奇、冒险、适合主题内容延展；C. 专业守护：可靠、高效、适合商业服务场景。
- 用途问：您希望先生成哪个范围？A. 完整 IP 系统：主视觉 + 设定 + 三视图 + 应用；B. 核心物料：主视觉 + 设定 + 3-5 张关键应用；C. 先看方向：先出 1 张主视觉确认角色。
- 风格问：你更倾向哪种视觉风格？A. 软萌潮玩：圆润比例、可爱表情、3D 公仔质感；B. 精致商业：统一品牌色、细腻材质、成熟商业画面；C. 强叙事海报：动态动作、丰富场景、标题压版。

必须先澄清的情况：四项关键信息缺失 3 项及以上、提及参考xx但未上传图片、用户指定资产之间明显冲突。


## 范围与差异

完整 VI 方案中默认包含 3-4 张 IP 图像，除非 IP 是明确的主需求。一个均衡的 IP 套组包括：

1. 主视觉或设定卡，
2. 三视图或角色设定圣经页，
3. 动作/场景海报，
4. 有需要时加入表情包/周边/应用图。

当用户要求“像完整 IP 提案”“像作品集长图”“参考图那种效果”时，不要只生成单张角色图。应把成套 IP 设计拆为可执行交付物：顶部世界观主视觉、用户指定的完整主标题/Logo 区、角色故事短文、草图/过程/建模展示、白底设定页、三视图、多动作姿态、主题场景卡、应用物料或周边展示。根据用户预算和数量要求合并为 3-6 张关键图，或生成一张纵向提案长图。

主题套图适配多主题。可根据品牌和用户输入扩展为节日、城市、航天、自然、科技、食品、文旅、教育、运动、公益、音乐、游戏、零售活动等主题；主题变化只能改变场景、道具、动作、服装局部和传播物料，不能改变 IP 的身份锚点。

如果 IP 只是 Logo、包装或活动需求的辅助元素，使用 1-2 张图即可。如果两个输出在姿势、镜头和用途上相似度超过 70%，则合并它们，或让其中一个变得明显不同，例如清晰的建模设定页与氛围型场景海报。

## 交付物知识库条目

交付时使用稳定的资产条目 ID、中文标题、用途说明和推荐画幅；真实图片 URL、文件对象或图片对象属于工具结果，不作为知识库条目名称。

| 资产条目 ID | 知识库标题 | 用途 | 推荐画幅 |
|---|---|---|---|
| `ip-main-view` | IP 主视觉 | 官方主视觉 / 标准姿势 | 16:9 或 4:5 |
| `ip-character-bible` | 角色设定圣经页 | 特征、比例、材质、道具 | 16:9 |
| `ip-setting-card` | 角色设定卡 | 名称、角色、性格、口头禅、故事 | 16:9 或 4:5 |
| `ip-turnaround` | 角色三视图 | 同一角色的正面/侧面/背面 | 16:9 或 4:3 |

## 一致性规则

锁定角色物种/身体、头部轮廓、脸部结构、眼睛风格、服装逻辑、主配饰、道具逻辑、材质和主配色。只有姿势、表情、镜头角度、场景、光照和应用形式可以变化。

三视图指同一个角色的多个角度，不是三个不同角色。

多图或长图输出时，所有画面都必须共享同一身份锚点：相同头身比、头部轮廓、脸部结构、眼睛形状、嘴鼻位置、标志服装、主配色、材质质感和核心道具。主题套图可以替换局部配件和场景道具，但不能把角色变成另一个物种、另一个年龄段或另一套主服装。

服装版本设定只能改变服装款式、局部配饰、编号、色卡和辅助配色。必须保持同一个角色的脸、头身比、物种、眼睛、嘴鼻、核心材质、性格和主识别点，不能因为换装而变成新角色。

## 提示词写作规则

每个资产条目都必须输出 `tool_mode`、`image_reference_url_list`、`asset_id`、`title`、`prompt`、`visible_text` 和推荐 `ratio`。核心图 `tool_mode` 为 `core_gen`，生成后记录 CDN URL；所有衍生资产 `tool_mode` 必须为 `derivative_edit`，且 `image_reference_url_list` 必须包含 `core-ip` CDN URL。`prompt` 条目就是高密度的 `design_brief` 本身，会直接发送给图像工具。模型、尺寸、种子、文件路径和脚本参数属于 manifest 或工具字段；不要把它们放进提示词正文。

如果使用多张参考图，用 `[img0]`、`[img1]`、`[img2]` 引用。每张图都必须独立描述：它提供了哪个角色/形态/风格、必须保留什么、当前交付物会如何使用它。不要把多图参考概括成“相同风格”或“所有参考”。

参考图只转化为具体规则：角色比例、渲染质感、版式层级、图文关系、色彩系统、场景构成、应用类型和成套感。不要在提示词中写“参考图效果”“类似参考图”“同款风格”作为替代描述。

成套 IP 提案应按交付物明确画面层级：主视觉使用大场景和角色核心姿态；设定页使用白底或浅底、正侧背视图、比例辅助线和标签；动作页使用多姿势网格或横向阵列；过程板展示草图、配色块、建模灰模或材质样；社媒场景卡使用多张带界面的卡片拼贴；主题套图围绕同一主题展开多场景应用；提案长图按从上到下的阅读顺序组织主视觉、叙事、过程、设定和应用。

`ip-main-view` 和 `ip-setting-card` 必须明显区分。`ip-main-view` 是传播主图，突出品牌理念、角色气质、主题场景、视觉冲击、主标题/Logo 和背景氛围，可用中心大角色、斜向构图、标题压版、场景纵深或强对比背景色。`ip-setting-card` 是信息设定页，突出角色姓名、身份、性格、关键词、口头禅、能力/道具、色卡、小图标、局部特写和信息模块，可用左右分栏、模块卡片、标签注释、浅底规范页或局部放大。两者至少在版式结构、背景色、信息密度、镜头距离、文字层级、配色策略中变化 3 项；禁止都使用白底、正面站姿、少量文字的同质方案。

`ip-turnaround` 和 `ip-setting-card` 不能只描述角色本体，必须同步构思配色、排版、布局、字体风格、背景处理和信息层级。三视图可使用网格底、圆角展示框、顶部标题区、视图标签、比例辅助线、色卡、底部说明和统一字体；设定卡可使用左右分栏、信息模块、关键词标签、局部放大、道具拆解、小图标和角色小姿态。

`ip-scene-poster` 必须参考用户参考图中吸睛、丰富、强叙事的海报效果：大场景、强光影、前中后景层次、主题道具、环境细节、运动感和清晰视觉中心。角色不能只是白底站立或正面摆拍，必须有明确动作，例如奔跑、跳跃、驾驶、探索、挥手、飞行、攀爬、发现目标、使用道具、与环境互动或执行任务。海报需要写明镜头设计，例如低角度仰拍、广角近景冲击、斜构图、景深、速度线、尘土/水花/光轨/烟雾等动态效果。场景可以丰富，但角色必须占据主视觉中心或强焦点位置。

`ip-scene-poster` 和 `ip-concept-poster` 都必须重视文字排版。每个海报 prompt 必须写清主标题、副标题、品牌名、日期/编号/角标等文字的完整内容，并全部使用中文双引号包裹；同时描述字体风格、字号层级、位置、对齐、颜色、描边/投影/压版方式和与角色的遮挡关系。角色海报强调角色视觉中心和传播冲击；概念海报强调世界观、叙事场景、氛围、空间纵深和探索感。

`ip-concept-poster` 应围绕“角色与世界的关系”构思，而不是单纯展示角色站姿。先确定一个叙事瞬间：角色正在经历、发现、选择、对抗、抵达或离开什么；再用景别、动作语言、构图和氛围共同表达这一瞬间。景别用于建立多样性：特写强调情绪，近景强调动作，中远景强调角色与场景关系，大全景强调世界观尺度。动作语言应来自剧情目标、场景压力或角色性格，而不是默认站立展示。氛围设计必须服务叙事，光影、天气、材质、空间、色彩应共同指向同一种情绪。构图可以让角色处于中心、边缘、背影、半遮挡、小比例或强透视位置，但必须有明确视觉焦点。

`ip-scene-poster` 和 `ip-concept-poster` 都必须有景别、镜头、动作语言和空间关系设计。同一套方案中不要连续使用相同的中景站姿；至少说明角色处于什么景别、正在做什么、与环境或道具发生什么关系、画面情绪由哪些光影和空间元素共同建立。

`ip-expression-sheet` 和 `ip-sticker` 必须有版式设计。可选九宫格、四宫格、标题区 + 网格、横向条或分组卡片；每个表情必须有不同情绪、动作符号、局部道具或背景色块，不能只是换嘴型。若出现标题、情绪标签或气泡文字，必须逐字写出并用中文双引号包裹。

`ip-merchandise-set` 必须明确不同产品的展示排列方式，不得只写“周边套装展示”。至少说明产品清单、主次关系、排列方式、视角、背景、标签文字和品牌图案使用位置。常用排列：中心主物 + 环绕配件、2x3 或 3x3 网格陈列、俯视桌面平铺、货架/陈列台、生活方式场景、包装开箱套组。至少包含 3 类不同产品，除非用户指定单品；同一套周边必须保持 IP 图案、品牌色、材质和包装语言一致。若有产品标签、价格牌、包装文字，必须逐字写出并用中文双引号包裹。

结构、色彩、道具、UI、包装类资产必须服务于复现和落地。`ip-construction-sheet` 强调比例和结构；`ip-color-material-board` 强调色彩、材质和纹理；`ip-prop-sheet` 强调道具结构和使用方式；`ip-ui-state-set` 强调产品界面中的空/加载/成功/错误/客服状态；`ip-packaging-application` 强调包装层级、开箱关系和印刷文字；`ip-campaign-key-visual` 强调活动传播、强标题和强场景。

影视角色设计类资产必须重视角色弧光、身份背景、服装逻辑、材质真实感、表演状态和镜头语言。影视场景不强制使用 Logo、品牌色和周边语言；可转为服装、道具、剧情时刻、场景氛围和制作参考。仍必须基于同一个 `core-ip` CDN URL 使用 edit / 图生图，保持角色身份一致。

整套 IP 方案必须有多样性。除角色身份锚点外，每张图至少变化 2-3 个维度：背景色或场景色、角色动作、镜头远近、构图方向、道具互动、光影氛围、信息密度、应用场景、服装局部、传播语气。不要连续输出背景色、动作、构图、镜头和场景氛围高度相似的图片。

### 文字渲染硬约束

所有要渲染的文字必须详细明确，并同时出现在 `visible_text` 与 `prompt` 正文中。每一处文字都必须使用中文双引号包裹，例如：“星球探索计划”、“IP settings”、“Scene poster”、“2025 Design by saaam”。

每条文字描述必须包含：完整文字内容、语言、所在位置、层级用途、字体风格、颜色、对齐方式和视觉处理。不要写“品牌标题”“英文小字”“Logo 文案”“说明文字”“手写标注”“一些标签”这类模糊文字指令。

`visible_text` 使用逐条清单，格式建议为：`位置 / 用途 / 完整文字 / 字体与颜色`。如果不希望画面出现文字，`visible_text` 写 `画面不出现任何文字`，并在 prompt 正文重复这一句。

禁止生成未声明文字、乱码、伪字、随机英文、随机日期、随机署名、占位符、错别字和模型自行补充的装饰性文字。若需要 Logo，只能渲染用户指定的完整 Logo 文字或明确写出的符号说明。

### 文生图prompt

文生图 prompt 只适用于 `core-ip` 核心形象或核心候选。对于新的核心 IP 图像，按以下顺序撰写提示词：

1. **总览**：第一句话说明图像类型、载体/版式，以及 IP 渲染情绪。可选第二句话说明背景/材质和角色概念。
2. **分区扫描**：按从上到下、由中心向外，或正面/侧面/背面的顺序描述。每个可见元素都需要说明绝对位置、元素类型、形态细节、中文双引号内的文字内容、文字风格、状态、对齐方式、颜色/材质和视觉处理。
3. **条件收束**：对于简单角色设定页，补充 1-2 句全局配色、材质和性格说明。
4. **画幅比例**：按“比例和尺寸决策”选择比例，并以 `画幅：{ratio}` 结尾。

### 图生图prompt

对于所有基于 `core-ip` CDN URL 的 IP 衍生图，按以下顺序撰写提示词：

1. **生成目标**：第一句必须说明 `使用 image_reference_url_list 中的 core-ip CDN URL 作为唯一角色参考`，再说明要生成的新 IP 交付物。例如：`使用 image_reference_url_list 中的 core-ip CDN URL 作为唯一角色参考，提取【角色体型、脸部结构、主色、标志道具】四类角色要素，生成【三视图设定页】。`
2. **角色要素提取**：只列出可复用的角色锚点：物种/身体、头身比、脸/眼/嘴、服装逻辑、标志性道具、材质质感、配色和身份识别点。不要描述源图背景、裁切、海报文字、镜头光照或偶发道具，除非它们定义了角色。
3. **目标画面展开**：按物理顺序描述新的主视觉、设定圣经页、三视图、动作延展、场景海报、表情包、周边或 IP-Logo 关系。提示词的大部分内容应定义目标姿势、视角、表情、动作、场景、设定页版式和可见标签。
4. **一致性与质量约束**：只保持角色锚点一致，同时允许新的姿势、表情、镜头角度、背景、动作和应用形式。明确渲染风格、材质、轮廓清晰度、表情质量和跨图像可识别性。
5. **画幅比例**：按“比例和尺寸决策”选择比例；如需保持参考图比例，写明 `画幅：沿用参考图比例`，否则以 `画幅：{ratio}` 结尾。

### 提示词条目

提示词条目是直接用于对应工具调用的指令。必须保留上方完整的文生图或图生图结构，不要使用缩短版二级提示词；衍生资产只能进入 edit / 图生图调用。

- 每个 IP 交付物都必须独立唯一；不要把同一个提示词复用于主视觉、设定圣经页、三视图、动作、场景、表情包和周边。
- 描述参考特征，而不是路径。多参考提示词必须分别命名 `[img0]`、`[img1]`。
- 先说明要生成什么，再说明从参考图中提取哪些角色元素。避免过度保留源图版式、背景、裁切、光照、装饰性文字或无关场景细节。
- 列出所有预期渲染出来的文字，并使用中文双引号，例如 `“角色名”`。文字必须是完整内容，不能写成文字类别或占位符。如果不应出现文字，写 `画面不出现任何文字`。
- 将所有负面约束集中放在最后一行 `禁止：...`。
- 省略像素尺寸、模型名称、种子、CLI 参数、输出文件名、图片 URL 和源路径。提示词最后一行只保留 `画幅：{ratio}` 或 `画幅：沿用参考图比例`。
- 衍生资产条目必须显式写 `tool_mode: derivative_edit` 和 `image_reference_url_list: ["{core-ip CDN URL}"]`；核心图条目才允许 `tool_mode: core_gen`。

### 常用提示词骨架

`ip-system-longform`：从上到下描述纵向提案长图。顶部为世界观主视觉和角色核心姿态；中段为用户指定的完整主标题、Logo、短故事、过程图、草图或建模展示；下段为设定页、三视图、多动作、主题场景卡和应用物料。每个分区都要写明背景、角色位置、图文层级和完整可见文字。

`ip-main-view`：描述一张传播主视觉。先写品牌理念和角色使命，再写主场景、角色核心姿态、镜头距离、背景氛围、主标题/Logo 和视觉焦点；版式应偏强识别、强情绪、强传播，不能写成信息设定页。

`ip-setting-card`：描述一张角色信息设定卡。先写角色姓名、身份、性格关键词和设定模块，再写角色展示、局部特写、道具拆解、色卡、标签、小图标和信息分区；版式应偏清晰、丰富、可读，不能只是主视觉换背景。

`ip-turnaround`：描述同一角色的正面、侧面、背面，使用统一比例尺、网格底或浅色背景、圆角展示框、顶部标题区、视图标签、比例辅助线和底部说明。标签文字必须明确，例如：“Front”、“Side”、“Back”或用户指定中文标签；同时写清配色、字体风格、布局间距和背景纹理。

`ip-expression-sheet`：描述四宫格或九宫格表情展示页。顶部可有标题区，主体使用 2x2 或 3x3 网格，每格表情的情绪、眼睛、嘴型、动作符号、气泡、背景色块都要不同；若有情绪标签，必须写出完整文字并用中文双引号包裹。

`ip-action-extension`：描述同一角色的 6-9 个动作姿态，使用等距留白、统一光源、统一材质和可识别表情。每个动作只改变姿势和道具互动，不改变角色身份。

`ip-costume-variation`：描述同一角色的 3-5 个服装版本，参考服装设定图的横向阵列方式。每个版本必须保持同一脸型、头身比、物种和核心表情，只改变服装款式、局部配饰、辅助配色、编号和色卡；每个版本需要标明完整编号文字，例如：“#01”、“#02”、“#03”。若需要系列标题，必须写出完整内容，例如：“DRESS UP”。

`ip-social-scene-cards`：描述多张社媒卡片拼贴，每张卡片有独立场景、镜头、角色动作和卡片界面文字。卡片上的品牌名、标题、角标、手写标注都必须逐条声明。

`ip-merchandise-set`：描述一张周边应用展示图。先列产品清单，再选择排列方式：中心主物 + 环绕配件、网格陈列、桌面平铺、货架/陈列台、生活方式场景或包装开箱套组。必须写清哪个产品在中心、哪个在左右、哪个在前后排或网格位置；说明视角、背景、材质、品牌图案位置、包装文字和标签文字。

`ip-construction-sheet`：描述结构比例图。写清头身比、身体几何结构、比例辅助线、正侧参考、关键尺寸关系和可复现标注。

`ip-color-material-board`：描述色彩材质板。写清主色、辅助色、点缀色、材质样本、纹理、光泽、品牌色映射和使用比例。

`ip-prop-sheet`：描述道具拆解页。写清道具名称、结构分解、材质、正侧视图、使用方式、和角色身体/动作的关系。

`ip-ui-state-set`：描述 UI 状态应用。展示空状态、加载、成功、错误、客服提示等 4-6 个状态，每个状态保持同一角色锚点，但表情、动作和提示语不同。

`ip-packaging-application`：描述包装应用图。写清礼盒、盲盒、吊牌、贴纸封口、说明卡、内衬、开箱层级和包装文字的排列方式。

`ip-campaign-key-visual`：描述活动 KV 主视觉。写清活动主题、主标题、角色动作、强场景、传播语、品牌识别和文字压版层级。

`ip-cinematic-character-concept`：描述影视角色概念图。写清角色身份、时代/世界观、服装轮廓、材质、表演状态、镜头景别和第一印象。

`ip-costume-prop-breakdown`：描述影视服装道具拆解。写清服装层次、面料、磨损、装备位置、标志道具、道具功能和局部特写。

`ip-cinematic-expression-pose`：描述影视表演表情与姿态。展示 4-6 个表演状态，强调情绪强度、眼神、肢体语言和角色弧光。

`ip-scene-moment-board`：描述剧情场景瞬间。写清关键剧情时刻、角色与环境关系、镜头语言、光影氛围、动作和情绪转折。

`ip-production-reference`：描述制作参考页。面向美术、建模、服化、分镜使用，写清结构、服装、道具、材质、色彩、表情、姿态和禁用项。

`ip-scene-poster`：描述一张强传播感角色海报。先写主题场景和叙事目标，再写景别、镜头、角色动态动作、表情、道具互动、前景/中景/背景、光影氛围和完整可见文字。必须比设定页更戏剧化、更丰富、更有行动感；禁止默认白底站姿、单一正面站立、空背景、弱动作、弱场景。

`ip-concept-poster`：描述一张世界观概念海报。先写角色与世界的关系、主题概念和叙事瞬间，再写景别选择、角色动作语言、空间尺度、空间纵深、光影天气、材质色彩和情绪氛围；文字排版要参与构图，完整说明主标题、副标题、品牌名、编号/日期/角标，并全部使用中文双引号包裹。

`ip-theme-suite`：围绕一个主题生成主视觉、场景应用、物料展示和角色动作组合。主题只影响场景、道具、局部服装和传播语，不改变角色主设定。

### 红线检查

最终确认前，检查：没有受保护 IP、公众人物或艺术家名称，除非用户提供；没有敏感政治/宗教/性/暴力内容；没有不适当的身体暴露；没有 `xx` 或 `某某` 之类占位符；所有渲染文字都已完整写出并加中文双引号；没有“标题文字”“英文小字”等模糊文字；图生图简报完整保留了必需的角色锚点。

工具调用前检查：核心图是否已存在为 `core-ip` CDN URL；所有衍生资产是否都是 `tool_mode: derivative_edit`；所有衍生资产是否都带 `image_reference_url_list: ["{core-ip CDN URL}"]`；是否存在衍生资产仍为 gen，如果有则禁止执行。

## QA

- 每张图中的角色都保持可识别。
- 每个提示词都有面向该交付物的专属场景/动作/视角指令。
- 主视觉、设定圣经页、三视图、动作、场景、表情包和周边共享同一身份。
- 图内文字要短，但必须逐字明确；较长的设定说明应放在指南中。
- `visible_text` 与 `prompt` 中声明的文字一致，且每个文字内容都用中文双引号包裹。
- 不存在未声明文字、乱码、伪字、随机英文、随机日期或随机署名。
- 场景海报可以丰富，但不能淹没角色。
- 角色海报和概念海报必须有明确文字排版策略，所有海报文字都逐字写出并用中文双引号包裹。
- 角色海报和概念海报必须有景别、镜头、动作语言和空间关系设计，概念海报要体现角色与世界的关系，不能退化为角色站在背景前。
- 三视图、设定卡、表情包必须包含配色、排版、布局、字体风格和背景处理，不只描述角色本体。
- 表情包优先使用四宫格或九宫格等明确版式，每格情绪和动作符号不同。
- 周边应用展示必须明确不同产品的排列方式、主次关系、视角、背景和标签文字，不能只写“周边套装展示”。
- 结构、色彩、道具、UI、包装类资产必须能支持复现或落地应用，不只做氛围图。
- 影视角色设计必须保持角色弧光、身份背景、服装逻辑、材质真实感、表演状态和镜头语言一致。
- 主题套图只改变主题场景和应用方式，不改变 IP 的身份锚点。
- 姿态扩展组只选择 1-2 个最贴合用户意图的类别，不默认全量输出。
- 整套图片在背景色、动作、镜头、构图、光影、场景和应用目的上有明确差异，不能同质化。
- `ip-main-view` 与 `ip-setting-card` 至少在版式、背景色、信息密度、镜头距离、文字层级、配色策略中变化 3 项。
