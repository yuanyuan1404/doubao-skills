## Word 文件回填（核心流程）

用户给了 Word 简历/模板，内容优化后需要回填。这是最常见的场景。

### 三步走

**第一步：解包**

```bash
uv run scripts/docx_edit.py unpack resume.docx unpacked/
```

功能：
- 解压 .docx 为 XML 文件目录
- 自动 pretty-print XML（方便阅读和编辑）
- 合并相邻的同格式 run（解决 Word 把文本拆成碎片的问题）
- 简化修订标记
- 转义智能引号为 XML 实体

解包后，核心内容在 `unpacked/word/document.xml`。

**第二步：编辑 XML**

在 `document.xml` 中找到要修改的文本，用编辑工具直接做字符串替换。

**做什么：**
- 找到原始文本所在的 `<w:t>` 元素
- 将 `<w:t>` 中的内容替换为优化后的文本
- 保持周围的 `<w:rPr>`（格式）和 `<w:pPr>`（段落属性）不变

**不要做什么：**
- 不要删除或修改 `<w:rPr>` 中的格式信息
- 不要写 Python/JS 脚本来操作 XML（容易破坏结构）
- 不要改动 `<w:sectPr>`（页面设置）

**第三步：打包**

```bash
uv run scripts/docx_edit.py pack unpacked/ resume_updated.docx
```

功能：
- 压缩 XML 空白
- 自动验证（与原文件对比，检查结构完整性）
- 自动修复常见问题
- 打包为 .docx

如果只是简单文本替换不需要验证：
```bash
uv run scripts/docx_edit.py pack unpacked/ resume_updated.docx
```

## XML 编辑详细规范

### 基本结构

Word 文档的文本结构：

```xml
<w:body>
  <w:p>                          <!-- 段落 -->
    <w:pPr>                      <!-- 段落属性（对齐、间距、样式等）-->
      <w:pStyle w:val="Heading1"/>
    </w:pPr>
    <w:r>                        <!-- 文本运行（run）-->
      <w:rPr>                    <!-- 运行属性（字体、大小、加粗等）-->
        <w:b/>                   <!-- 加粗 -->
        <w:sz w:val="24"/>       <!-- 字号 -->
      </w:rPr>
      <w:t>这里是文本内容</w:t>   <!-- 实际文本 -->
    </w:r>
  </w:p>
</w:body>
```

### 简单文本替换

最常见的操作。找到 `<w:t>` 元素，替换其中的文本：

**修改前：**
```xml
<w:r>
  <w:rPr><w:b/><w:sz w:val="22"/></w:rPr>
  <w:t>负责后端开发工作</w:t>
</w:r>
```

**修改后：**
```xml
<w:r>
  <w:rPr><w:b/><w:sz w:val="22"/></w:rPr>
  <w:t>主导搭建订单处理微服务，日均处理 800 万笔订单</w:t>
</w:r>
```

注意 `<w:rPr>` 完全没动——格式保留。

### 替换整个 run

当需要改变 run 的边界（比如原来一个 run 拆成两个，或合并两个 run）时：

**规则：**
- 复制原始 `<w:rPr>` 到新 run 中
- 如果文本包含空格开头或结尾，添加 `xml:space="preserve"`

```xml
<w:r>
  <w:rPr><w:b/><w:sz w:val="22"/></w:rPr>
  <w:t xml:space="preserve">优化后的文本 </w:t>
</w:r>
```

### 特殊字符处理

| 字符 | XML 实体 |
|------|---------|
| 左双引号 " | `&#x201C;` |
| 右双引号 " | `&#x201D;` |
| 左单引号 ' | `&#x2018;` |
| 右单引号 ' | `&#x2019;` |
| & | `&amp;` |
| < | `&lt;` |
| > | `&gt;` |

`docx_edit.py unpack` 已经把智能引号转成了 XML 实体，编辑时保持用实体。

### 添加新的列表项

如果需要在已有列表中添加新的项目符号条目：

1. 找到已有的列表项 `<w:p>`
2. 复制整个 `<w:p>` 元素（包括 `<w:pPr>` 中的 `<w:numPr>`）
3. 只修改 `<w:t>` 中的文本

这样新条目会继承相同的列表样式和缩进。

### 删除内容

直接删除整个 `<w:p>` 段落即可。不要只清空文本——那会留一个空行。

## 生成全新 Word 文件

当用户没有现有文件、需要从零生成时，用 docx-js（Node.js 库）。

### 安装

```bash
npm install docx
```

### 核心规则

这些规则违反了会导致 Word/WPS 显示异常：

**页面设置：**
- A4：width=11906, height=16838（DXA 单位）
- 2cm 边距：1134 DXA
- 字体：中文"微软雅黑"或"思源黑体"，英文 Arial

**项目符号（最容易出错）：**

```javascript
// 错误 — 手动插入符号字符
new Paragraph({ children: [new TextRun("• 成就描述")] })

// 正确 — 使用 numbering 配置
const doc = new Document({
  numbering: {
    config: [{
      reference: "bullets",
      levels: [{
        level: 0,
        format: LevelFormat.BULLET,
        text: "\u2022",
        alignment: AlignmentType.LEFT,
        style: { paragraph: { indent: { left: 720, hanging: 360 } } }
      }]
    }]
  },
  // ...
});

new Paragraph({
  numbering: { reference: "bullets", level: 0 },
  children: [new TextRun("成就描述")]
})
```

**表格（第二容易出错）：**

```javascript
// 必须同时设置 columnWidths 和每个 cell 的 width
new Table({
  width: { size: 9638, type: WidthType.DXA },
  columnWidths: [4819, 4819],    // 各列宽度，加起来等于总宽度
  rows: [
    new TableRow({
      children: [
        new TableCell({
          width: { size: 4819, type: WidthType.DXA },  // 必须和 columnWidths 对应
          // ...
        })
      ]
    })
  ]
})
```

**其他：**
- 不用 `\n` 换行，用独立的 Paragraph
- 表格着色用 `ShadingType.CLEAR`（不是 `SOLID`）
- 宽度只用 `WidthType.DXA`，不用百分比

## PDF 生成

使用 Typst 排版引擎。

```bash
# 安装
brew install typst

# 编译
typst compile resume.typ resume.pdf
```

### 中文字体

```typst
#set text(font: ("Inter", "Noto Sans SC"))        // 思源黑体
#set text(font: ("Arial", "PingFang SC"))          // 苹方（macOS）
```

### 基本模板

```typst
#set page(paper: "a4", margin: (top: 2cm, bottom: 2cm, left: 2cm, right: 2cm))
#set text(font: ("Inter", "Noto Sans SC"), size: 10pt)

#align(center)[
  #text(size: 18pt, weight: "bold")[张三]
]

#align(center)[
  138-0000-1234 | zhangsan\@email.com | 北京
]

#let section-title(title) = {
  v(8pt)
  text(size: 12pt, weight: "bold")[#title]
  line(length: 100%, stroke: 0.5pt)
  v(4pt)
}

#section-title("工作经历")
```

## 常见格式问题

### 替换文本后格式丢失

**原因：** 连同 `<w:rPr>` 一起替换了
**修复：** 只替换 `<w:t>` 中的文本，保留 `<w:rPr>` 不动

### 占位符被拆成多个 run

**原因：** Word 内部经常把连续文本拆分（比如拼写检查后）
**修复：** 手动合并相邻的同格式 run（将多个 `<w:r>` 的文本合到一个里，保留一份 `<w:rPr>`）

### 打包后 Word 报错

**原因：** XML 结构被破坏（标签未闭合、元素顺序错误等）
**修复：** 对比修改前后的 XML，检查是否误删了结构元素（如 `<w:sectPr>`、`<w:tblPr>` 等）

### 新增内容导致超页

**修复：**
1. 先调整间距（行高、段间距、边距）
2. 精简内容
3. 缩小字号（不建议低于 10pt）
