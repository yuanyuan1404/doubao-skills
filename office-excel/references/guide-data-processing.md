# 数值计算与处理（caculating-skills）

本技能为处理电子表格（如 Excel）等结构化数据的复杂计算任务，提供从需求理解到结果交付的完整工作流程与执行标准。核心目标是确保计算逻辑准确、过程稳健、交付产物专业合规。

## 一、意图理解与主体映射

准确计算的前提是深度理解需求，而非简单匹配关键词。

1. **需求与数据双向校准**：启动计算前，必须将用户文本需求与提供的数据样本进行双向验证：
   - 仔细分析需求，明确核心分析目标（即"主体"）
   - 检查数据列名和示例行，理解每个字段的实际业务含义
   - 确保需求与数据结构匹配，避免基于错误理解开始计算

2. **精准识别分析主体**：当数据中存在多个相似但角色不同的字段时，必须结合上下文语境，将分析主体准确映射到对应字段。此步骤是确保后续分析围绕正确对象展开的基础。

## 二、数据预处理

所有计算必须基于干净、规范的数据。应用公式前，必须完成系统性数据预处理。

1. **数据预检（Excel附件专用）**：
   - 当用户上传的附件为Excel文件（.xlsx, .xls）时，首先运行预检脚本获取数据结构信息
   - 预检脚本位置：`scripts/inspect_workbook.py`
   - 运行命令：`python scripts/inspect_workbook.py <excel文件路径>`
   - 预检输出包含：所有sheet名称、每个sheet的行数和列数、前15行原始数据预览、合并单元格范围、每列数据类型分布及缺失值统计、疑似合计/小计等特殊行（`special_rows`）

2. **识别并处理特殊汇总行**：
   - 识别包含“合计/总计/小计/汇总/累计”等关键词的行，并结合数值分布确认是否为汇总口径
   - 明细分析/聚合/建模时默认排除这些汇总行，避免重复统计；需要保留时单独输出并注明口径

3. **格式与类型统一**：
   - 检查参与计算的列，确保数据类型一致
   - 将文本格式数字转换为数值类型
   - 统一不同写法的日期为标准日期时间对象
   - 对包含非数值字符（如单位"个"、"元"）的列，先提取数值部分再计算

4. **异常与缺失值处理**：
   - 识别数据中的空值（`null`）、错误标记（`#N/A`）等异常条目
   - 根据业务逻辑采取恰当策略：
     - 用中性值（如 0 或平均值）填充
     - 聚合计算中直接忽略异常记录
     - 对异常值进行标记并在最终结果中说明

5. **复合字段拆分**：
   - 主动识别并解析包含多个信息点的单一字段
   - 例如，将时间范围字段（如 `09:00-18:00`）拆分为"开始时间"和"结束时间"两个独立字段
   - 确保拆分后的字段可直接用于后续计算

## 三、时间与日期处理

时间与日期计算易因格式和逻辑问题出错，需严格遵循以下处理原则。

1. **使用标准化时间对象**：
   - 执行任何时间相关运算（如求差、比较）前，确保所有操作数已转换为标准日期时间对象
   - 从根本上避免因文本格式不一致导致的计算错误

2. **区间解析优先**：
   - 遇到表示时间段的文本时，严格遵循"先拆分，再计算"原则
   - 将时间段解析为两个独立时间点后，再进行持续时间或间隔计算

3. **输出人类可读格式**：
   - 生成最终交付物时（特别是写入 Excel），确保所有日期时间字段以人类可读格式呈现（如 `YYYY-MM-DD HH:MM`）
   - 避免使用 Excel 原始日期序列号格式（即距离基准日期的天数数值）
   - 如需格式转换，可参考 `references/ref-xlsx-workflow.md`

## 四、公式设计与边界处理

稳健的计算逻辑需同时覆盖常规场景和边界情况。

1. **逻辑与业务口径对齐**：
   - 设计计算公式时，严格遵循已明确的业务口径
   - 对于比率类指标（如完成率、转化率），明确分子和分母的准确定义
   - 遇到特殊计算公式（如政策、金融相关），调用 `general_search` 函数查询实现细节

2. **分母为零的容错处理**：
   - 执行除法运算前，必须检查分母是否可能为零
   - 定义清晰的备用输出策略：返回 0、空值或特定标记文本（如 `N/A`）
   - 避免程序中断或产生无穷大等误导性结果

3. **空值与极值处理**：
   - 聚合计算（如求平均、加总）中明确空值处理策略：
     - 不影响计算和分析的空值视为 0
     - 影响计算和分析的空值，避免相关计算并在输出中说明空值情况及处理方案
   - 审视计算结果合理性，警惕极端值导致的结果失真

## 五、交付与安全

1. **结果与过程分离**：
   - 最终产物中不得包含任何内部实现过程痕迹
   - 过滤思考过程、工具调用日志、中间代码或命令
   - 仅呈现纯净、最终的计算结果或结论

2. **封装与可用性保障**：
   - 根据用户需求确定交付物形式
   - 所有需下载的产物（如报告、数据文件）通过规范途径封装
   - 确保交付链接有效，文件可正常打开使用

3. **格式专业性**：
   - 表格列宽适中
   - 数字、百分比和日期显示方式统一
   - 确保信息一目了然，易于理解
