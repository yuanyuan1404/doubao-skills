# 量化建模（data-modeling）

本技能从 Excel/CSV 数据构建量化模型，支持预测、评价、优化三类决策场景。

## 工作方式

分三步工作：理解问题 → 执行分析 → 给出结论。

### 第一步：理解问题（OFACE 框架）

收到请求后，先用 OFACE 框架快速梳理问题要素。用户不会补充信息，缺失项基于数据文件和常识做合理假设。

**O — Objective（目标）**：判断问题类型

| 类型 | 信号词 | 对应 reference |
|------|--------|---------------|
| 预测 | 预测、预估、趋势、未来、forecast | `references/modeling-prediction.md` |
| 评价 | 评价、排名、打分、评分、综合评估、选择最优 | `references/modeling-evaluation.md` |
| 优化 | 优化、最优、最大化、最小化、分配、规划 | `references/modeling-optimization.md` |

**F — Frame（对象与粒度）**：建模对象是什么，粒度如何（实体×时间×类别），有无排除规则。从数据的行列结构推断。

**A — Assets（数据与变量）**：先读取数据表头和前几行，了解数据概况（样本量、列名、数据类型、缺失情况），识别目标变量 Y 和特征 X。CSV 文件注意编码问题，优先尝试 utf-8-sig、gbk、gb18030。

**C — Criteria（评价标准）**：根据问题类型选默认指标——回归用 RMSE/R²，分类用 F1/AUC，评价用排名稳健性，优化用目标函数值。用户指定的优先。

**E — Explain（解释需求）**：用户想了解什么？因素重要性、分组差异、情景模拟？默认至少输出因素重要性排序。

梳理后，向用户简要输出：问题类型、关键假设列表、计划使用的方法。

### 第二步：执行分析

1. **读取对应的 reference 文件**，按其中的执行流程完成分析（见上表）
2. reference 提供的是执行步骤和方法指引，据此编写适配用户数据的 Python 脚本
3. 绘图时注意配置中文字体（如 SimHei、STHeiti、PingFang SC 等），确保中文标题和标签正常显示
4. 图表保存为 PNG 文件

### 第三步：给出结论

建模完成后，**必须基于结果提炼出可直接使用的业务结论**。这是整个分析中最重要的部分——用户要的不是表格和指标，而是"所以结论是什么"。

每个 reference 的最后一步"结论提炼"给出了该场景下需要回答的结论性问题，务必逐条回答。

回答时：
- **用业务语言回答**，不堆砌技术指标。例如"影响销量最大的因素是促销活动，其次是单价"而非只列出 feature importance 数值
- **结合图表解读**，指出图中的关键信息和值得注意的模式
- **给出具体数值**：预测值、排名得分、最优方案等都要给出具体数字，不要只说"较高""较好"
- **给出可操作的建议**，基于分析结果提供决策参考
- **列出假设清单**，标注哪些假设对结论影响较大

## 输出格式

默认用 Markdown 格式回复。如果分析中生成了图表需要在报告中展示，可以生成自包含的 HTML 文件，将图片用 base64 内嵌，用户浏览器打开即可查看。

## 输出质量要求

- 代码必须可直接运行，需要安装的包要在开头用注释列出
- 所有图表必须有中文标题、轴标签、图例
- 最终要**回答用户的问题**，不能只输出模型和数据就结束
