# 数据挖掘（data-digging）

本技能为数据挖掘任务提供完整的执行框架，指导从问题定义到结果交付的全过程，确保分析的准确性、深度和专业性。通过系统化的工作流程和质量控制机制，帮助分析师有效处理各类数据挖掘任务，从数据中发现有价值的模式、关系和洞察。

## 快速决策

在开始数据挖掘任务前，需要快速做出以下关键决策：

- **明确挖掘目标**：确定具体的分析方向，如相关性分析、预测建模、聚类分析等
- **识别数据需求**：明确所需的数据类型、规模和质量要求
- **选择方法路径**：根据目标和数据特征选择合适的挖掘方法
- **规划交付形态**：确定最终交付物的形式和内容要求

## 工作流程

### 1. 问题分析与目标定义

- **需求澄清**：深入理解用户的问题和需求，明确挖掘目标和预期输出
- **范围界定**：确定分析的时间范围、地域范围和数据范围
- **成功标准**：定义衡量挖掘结果成功的具体指标
- **约束识别**：识别时间、数据、技术等方面的限制

### 2. 数据获取与预处理

- **数据收集**：识别并获取所需的数据来源，评估数据质量和完整性
- **数据清洗**：处理缺失值、异常值和重复值，统一数据格式和单位，解决数据不一致问题
- **数据转换**：进行特征工程（创建衍生变量）、数据标准化/归一化以及必要的数据降维处理

### 3. 探索性数据分析

- **描述性统计**：计算基本统计量，了解数据的分布特征
- **可视化分析**：使用各种图表探索数据中的模式和异常
- **相关性分析**：识别变量之间的关系和依赖
- **数据质量评估**：评估数据对挖掘任务的适用性

### 4. 挖掘方法选择与实施

- **方法选择**：根据分析目标选择合适的挖掘算法，同时评估算法的假设条件和适用范围，考虑计算资源和时间限制

**按任务类型选择方法**：

| 任务类型 | 推荐方法 | 适用场景 |
|---------|---------|---------|
| 描述性挖掘 | K-means / DBSCAN / 层次聚类 | 发现数据自然分组 |
| 关联规则 | Apriori、FP-Growth | 发现数据间关联规则 |
| 异常检测 | Isolation Forest、LOF、3σ/IQR | 识别异常值 |
| 分类预测 | 随机森林、XGBoost、逻辑回归 | 预测离散标签 |
| 回归预测 | 线性回归、Ridge/Lasso、随机森林回归 | 预测连续值 |
| 时间序列 | ARIMA/SARIMA、Prophet、滞后特征+XGBoost | 预测时间序列数据 |
| 文本挖掘 | LDA/NMF（主题）、词典/SVM（情感分类） | 结构化文本语义 |

**按数据特征调整**：

| 数据特征 | 推荐调整 |
|---------|---------|
| 高维数据 | PCA 降维或 t-SNE 可视化，注意维度诅咒 |
| 不平衡数据 | SMOTE 过采样 或 `class_weight='balanced'`，调整评估指标 |
| 时序数据 | 必须按时间顺序拆分，禁止随机 shuffle |
| 稀疏数据 | 优先线性模型或树模型，防止过拟合 |

- **模型构建**：数据划分（训练集、验证集、测试集）、模型参数调优和交叉验证（K折交叉验证；时序用 TimeSeriesSplit）
- **模型评估**：

| 模型类型 | 评估指标 |
|---------|---------|
| 分类模型 | 准确率、精确率、召回率、F1、AUC-ROC |
| 回归模型 | RMSE、MAE、R²、MAPE |
| 聚类模型 | 轮廓系数、Davies-Bouldin 指数 |
| 关联规则 | 支持度、置信度、提升度 |

解释模型结果的业务含义，对比训练集 vs 测试集指标检查过拟合。

### 5. 结果验证与解释

- **结果验证**：验证结果的一致性和可靠性，进行敏感性分析，测试结果在不同数据集上的泛化能力
- **结果解释**：解释挖掘结果的业务意义，识别关键影响因素和模式，提供可操作的洞察和建议
- **不确定性分析**：评估结果的置信水平，识别潜在的偏差和风险

### 6. 可视化与报告生成

- **结果可视化**：选择合适的图表类型（如散点图、热图、树状图、网络图等），确保图表清晰、美观、信息丰富，避免图表文字叠加、布局混乱等问题
- **报告撰写**：确保结构清晰，逻辑连贯，内容深度足够，避免表面分析，按照要求展示必要的图表和数据
- **交付物准备**：准备主报告（包含方法、结果、结论）、数据文件（处理后的数据、模型输出）以及代码和文档（分析脚本、模型说明）

### 7. 质量控制与审核

- **数据准确性检查**：验证数据统计和计算的准确性，交叉验证关键结果，确保数据标识和标签正确
- **方法严谨性检查**：避免使用未经验证的假设，确保分析方法的科学性和合理性，提供方法选择的依据和说明
- **结果一致性检查**：验证结果与业务逻辑的一致性，检查不同分析方法结果的一致性，确保结论有充分的数据支持

## 常见问题及解决方案

### 数据准确性问题

- 建立数据验证流程，交叉检查关键数据；使用自动化脚本验证数据计算
- 对数据来源和处理过程进行详细记录；实施多级审核机制

### 分析方法问题

- 明确方法选择的依据和假设条件；使用多种方法进行交叉验证
- 进行模型评估和验证；参考领域最佳实践和标准方法

### 可视化与格式问题

- 遵循数据可视化最佳实践；使用专业的可视化工具和模板
- 确保图表元素清晰可读；统一文档格式和样式

### 内容质量问题

- 制定详细的分析计划和深度要求；多角度分析数据，挖掘深层次关系
- 结合业务知识提供有价值的洞察；提供具体、可操作的建议

## 质量要求

- **数据完整性**：确保数据完整、准确，处理过程可追溯
- **方法严谨性**：使用科学、合理的分析方法，避免主观假设
- **结果可靠性**：确保分析结果可靠、一致，有充分的数据支持
- **报告专业性**：报告结构清晰，内容深度足够，格式规范美观
- **交付完整性**：按照要求提供所有必要的交付物
