# 优化类建模 — 执行流程

按以下步骤完成优化建模。每一步都要针对用户的具体数据编写代码。

---

## Step 1：识别优化问题结构

从用户描述和数据中提取三要素：

| 要素 | 问题 | 示例 |
|------|------|------|
| 决策变量 | 需要确定什么？ | 各产品产量、各项目投资额 |
| 目标函数 | 优化什么？最大化还是最小化？ | 最大化总利润、最小化总成本 |
| 约束条件 | 有哪些限制？ | 资源上限、最低/最高产量、预算 |

## Step 2：判断问题类型并选择求解方法

```
目标函数和约束都是线性的？
├── 是 → 线性规划
│   ├── 决策变量连续 → LP（用 PuLP）
│   ├── 决策变量必须是整数 → IP（用 PuLP，cat='Integer'）
│   └── 混合 → MIP（用 PuLP）
└── 否 → 非线性优化（用 scipy.optimize.minimize）
```

多数业务问题（生产计划、资源分配、运输调度）都是线性规划。

## Step 3：建模求解

### 线性规划（PuLP）

建模步骤：
1. 创建问题：`prob = LpProblem("名称", LpMaximize 或 LpMinimize)`
2. 定义决策变量：`LpVariable("名称", lowBound=下限, upBound=上限)`
3. 添加目标函数：`prob += 目标表达式`
4. 添加约束条件：`prob += 约束表达式, "约束名称"`
5. 求解：`prob.solve(PULP_CBC_CMD(msg=0))`

关键注意：
- 用 `PULP_CBC_CMD(msg=0)` 抑制求解器输出
- 整数变量用 `cat='Integer'`，0-1 变量用 `cat='Binary'`
- 如果有"最低产量"和"最高产量"，直接设为变量的 lowBound 和 upBound
- 决策变量上界要同时考虑多个限制（如 min(最高产量, 月均需求)）

### 非线性优化（scipy.optimize）

```python
from scipy.optimize import minimize
result = minimize(目标函数, x0=初始点, method='SLSQP', bounds=变量范围, constraints=约束列表)
```

约束格式：`{'type': 'ineq', 'fun': lambda x: ...}` 表示 >= 0 的不等式约束。

## Step 4：结果分析

求解后必须检查：

1. **求解状态**：是否找到最优解（LpStatus 为 'Optimal'）
2. **各决策变量取值**：列表展示
3. **目标函数最优值**
4. **约束利用情况**：哪些约束是紧约束（资源用完了），哪些有富余（slack > 0）
5. **灵敏度分析**（按需）：关键参数变化 ±10%/20% 时最优解如何变化

## Step 5：必须产出

以下内容不可省略，即使用户没有逐条要求：

1. **数学模型描述**：用简洁的文字/公式说明目标函数和主要约束，让用户知道建了什么模型
2. **最优方案表**：各决策变量的取值和对应的目标函数最优值
3. **约束分析**：哪些约束是紧约束（瓶颈资源），资源利用率多少，还有多少余量
4. **敏感性说明**：关键参数（如某种资源上限、某产品利润率）变化时，最优解会如何变动

必出图表：
- **最优方案柱状图**：各决策变量取值
- **约束利用率图**：水平柱状图，紧约束标红

按需出图：
- 灵敏度分析折线图
- 帕累托前沿（多目标时）

## Step 6：结论提炼

基于优化结果，必须明确回答以下问题：

- **最优方案总结**：用一句话说清楚最优方案是什么、目标值是多少（如"最优生产方案下总利润为 58.3 万元"）
- **资源瓶颈**：哪些资源是瓶颈（紧约束）？如果放松瓶颈约束，目标值能提升多少？
- **方案特征**：最优方案有什么特点？哪些决策变量取了上限/下限？为什么？
- **替代方案建议**：如果用户对某些约束有调整空间（如增加预算、延长工期），给出"如果…则…"的对比方案
- **建议行动**：基于优化结果，用户最优先应该执行什么？

示例结论："最优方案总利润 58.3 万元：产品 A 生产 200 件、产品 B 生产 150 件、产品 C 不生产。瓶颈资源为机器工时（利用率 100%），原材料仅用了 72%。如果机器工时增加 10%，利润可提升至 63.1 万元（+8.2%），建议优先扩充产能。"

## 常见陷阱

- 无可行解通常是约束矛盾——检查数据或放松约束
- 无界解通常是漏了约束——检查是否遗漏上下限
- 决策变量上界要同时考虑多个限制（如 min(最高产量, 月均需求)）
- 目标函数和约束的单位必须一致
