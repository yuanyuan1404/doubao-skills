# 文本语义处理（excel-semantic-analysis）

本技能用于将 Excel 中的非结构化文本抽取为结构化字段，并按需进行标准化、归类与汇总处理。

**适用场景**：用户要求对某列文本提炼要点/观点/类型、贴标签、归类、标准化或统计汇总，仅限 `.xlsx/.xls`。

## 快速决策

- **明确目标与输出**：首先确定抽取目标（需要哪些字段）和输出形态（标签体系、汇总维度、输出格式）
- **执行流程**：严格在 Excel 内操作，遵循"预检 → 抽取 → 归类 → 汇总"的流程

## 工作流程

### 1. 预检与确定重点字段

- **执行预检**：运行 `scripts/inspect_workbook.py <excel_path> [output_json]` 脚本
- **分析结构**：
  - 查看输出中的 `sheet_names` 与各 sheet 的 `preview_rows`（前15行原始矩阵），不省略空行
  - 查阅 `merged_cells` 与 `merged_cells_by_row`，识别合并单元格对表头与数据区的影响
- **识别重点字段**：结合表头与前几条数据，识别与语义分析相关的重要列
- **确定处理范围**：根据用户需求，明确重点处理的字段/列/行/范围

### 2. 澄清需求与口径（贯穿全流程）

- **初始确认**：明确抽取字段（要点/观点/类型/实体等）、标签体系与汇总维度
- **过程校准**：中途归类或抽取时如出现异常/疑虑（边界样本、冲突标签、过高的"其他/未知"占比等），必须回到用户需求与实际数据，重新确认口径或调整规则
- **方案确认**：口径不明确时，提供候选方案并请用户确认（例如：分类维度、是否允许"其他/未知"）

### 3. 定位文本区域

- **明确区域**：确定表头行、目标文本列、数据起止行、异常行（空行/注释/合并单元格）

### 4. 抽取与清洗

- **保留原始**：保留原始文本列，新增"清洗后文本"列
- **噪声处理**：去除明显噪声（多余空格、重复分隔符、占位符），不改变文本含义

### 5. 语义抽取

- **字段抽取**：按需求抽取要点、主要观点、类型、实体、时间、地点、数值等字段
- **结构化输出**：对每行生成结构化字段，必要时保留"证据片段"

### 6. 标准化与归类（优先可解释，允许近似）

- **标准化处理**：对同义/近义表达进行标准化（如同义词映射、统一格式），可参考 `scripts/type_division.py` 进行归类
- **多层归类策略**：文本不规整时，避免"完全一致才合并"的硬规则，采用以下策略：
  - **规则/词典归类**：建立主题词典与关键词映射，将相关表达映射到统一类别
  - **相似度聚类**：对清洗文本进行向量化处理，基于相似度阈值进行合并或聚类，生成"近似一致"分组
  - **先标准化再计数**：去除可变实体，归一化数字与日期等变量，再进行近似合并与计数
- **归类原则**：归类规则需具备可解释性；无法判断时归入"其他/未知"，并在 notes 中说明原因

### 7. 统计与汇总

- **维度统计**：按用户指定维度进行计数、占比、TOPN、交叉汇总等统计
- **结果输出**：将汇总结果写入新 sheet 或独立区域，并在结果中说明口径

### 8. 可追溯输出

- **映射关系**：每条抽取结果必须保留原始行号/列号映射
- **输出内容**：包含原始文本、清洗文本、抽取字段、分类标签、汇总口径说明

**输出字段规范**（写入新 sheet 或追加列时使用以下字段）：

| 字段名 | 说明 |
|--------|------|
| source_sheet | 原 sheet 名 |
| source_row | 原始行号（Excel 1-based） |
| source_col | 目标文本列名或列号 |
| raw_text | 原始文本 |
| cleaned_text | 清洗后文本 |
| key_points | 要点摘要（可为空） |
| main_view | 主要观点/结论（可为空） |
| entities | 实体列表（可为空） |
| category | 一级分类 |
| subcategory | 二级分类（可为空） |
| labels | 多标签（用分隔符连接） |
| normalized_value | 标准化结果（可为空） |
| evidence | 证据片段（可为空） |
| notes | 备注/不确定说明（可为空） |

**汇总 sheet 示例字段**：category_count（按 category 计数）、category_label_pivot（category × labels 交叉汇总）、top_entities（实体 TOPN）

**口径说明**：在汇总结果中明确注明分类体系、同义词映射规则与未知类处理方式。

### 9. 处理说明与方法披露

- **文字说明**：除 Excel 结果外，必须提供文字说明或易读摘要，清晰说明处理过程
- **说明内容**：至少包含以下信息
  - 分类数据来源（基于哪些列/范围、是否跨列合并）
  - 分类方法（规则/词典/相似度聚类/人工确认等）
  - 分类逻辑（是否基于已有分类再细分，还是仅基于文本列重新划分）

## 质量要求

- **数据完整性**：只在用户要求的范围内修改；未要求的单元格/列/行不得改动。若用户要求保留原表，则在新 sheet 或副本中输出。
- **可追溯性**：结构化结果必须可回溯到原始行号/列号
- **准确性**：当规则不明确时，先提出澄清问题而不是猜测
